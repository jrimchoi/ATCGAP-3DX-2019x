File View

When you expand an ENOVIA vault and click a folder, the files and documents that are contained in that folder are listed in the File View in the right pane.

  • Folder List
  • File List



Folder List

The color of folders in Windows Explorer indicates their status in ENOVIA Collaborative Design for SolidWorks:

Green

The folder is in a vault and you have access to the vault database.
Blue

The folder is in a vault but you are off-line. Select Tools > Work On-line and log in.
Gray

The folder is local to your hard drive and does not exist in the vault database. Right-click and select Add to File Vault if you have permission to add files to the vault.
Yellow

The folder is not in your local view of the vault, and does not exist in the vault database.

File List

The files you have permission to access are displayed in the right pane.

The following column headings are standard for the file list and cannot be changed or deleted.

Name Name of the file or the folder.
Checked Out By The user who has checked out the file for editing. If blank, the file is available to all users with access rights.
Version File revision and version information.

In this example:


  • indicates that the local file is newer than the vault file.
  • A.2 means that the local file is revision A, version 2.
  • A.1 means that the vault file is revision A, version 1.
Size File size.
File Type The type of SolidWorks document.
State The current state of the file as defined in the lifecycle, for example, Initiated or Waiting for approval.
Modified Date the file was created or last modified.
Checked Out In Local path to the checked out file.

Note: You can drag column headers to change the order of the columns in the file list and click column headers to sort files.
