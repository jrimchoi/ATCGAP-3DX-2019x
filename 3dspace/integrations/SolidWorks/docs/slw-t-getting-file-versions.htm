Getting Versions

You can use the Get Version and Get Latest Version commands to copy a file version to your working folder.

You can open any version or revision of a file to recover it exactly as it was at that earlier stage. You can revert to this earlier version and make it the latest version, deleting all subsequent changes.

This task shows you how to:

  • Get File Versions in SolidWorks
  • Get File Versions in Windows Explorer

Get File Versions in SolidWorks

Opening a design in SolidWorks from the local vault view does not automatically retrieve the latest version of the design if an earlier version is already present in the local cache. The latest version is retrieved if the file has never been copied to the local cache.

Follow this procedure if you want to retrieve a specific version or the latest version of a design.

  1. With a file from the ENOVIA vault open in SolidWorks, do one of the following:

    • Click ENOVIA > Get.

      A Get dialog box opens, listing the available versions.

    • In the ENOVIA CommandManager or ENOVIA task pane, expand Get Version.

      A drop-down list shows the available versions.

  2. Select the version to get.

ENOVIA Collaborative Design for SolidWorks copies the master file from the vault to your local working folder (cache), overwriting the version in your cache. In the task pane, the Versions column shows the version status in your local cache and in the vault.

Get File Versions in Windows Explorer

You can open any version or revision of a file to recover it exactly as it was at that earlier stage. You can read the file version notes, roll back to this earlier version, and make it the latest version, deleting all subsequent changes.

  1. Select the file.

  2. Do one of the following:

    • Click Display > History. Select the version or revision and click Get.
    • Click Actions > Get Latest Version
    • Click Actions > Get Version to view any previous version or revision.

ENOVIA Collaborative Design for SolidWorks copies the master file from the vault to your local working folder (cache), overwriting the version in your cache. The Versions column in the file window shows the version status in your local cache and in the vault.
