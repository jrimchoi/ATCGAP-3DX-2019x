Saving Modified Designs to ENOVIA

This task shows you how to save a design to ENOVIA database that is modified.

Before you begin: Ensure that the modified design is open in AutoCAD.
Related Topics
About Saving in ENOVIA
Limitations in ENOVIA Save

  1. Click ENOVIA > Save from AutoCAD toolbar.

    The ENOVIA Save dialog box opens.

    Note: An error message is displayed and ENOVIA Save dialog box does not open if there are no changes to be saved in the designs open in AutoCAD.

    All the open designs in AutoCAD that are new or modified are listed in the ENOVIA Save dialog box. For each design, the ENOVIA Save dialog box lists,


    • . Displays the lock status of the design in ENOVIA. icon is displayed if the design is locked by you or any other user. icon is displayed if the design is not locked in ENOVIA. For a new design, no icon is displayed.
    • Name. Name of the design.
    • . This icon is displayed for designs that are saved to ENOVIA Workspace. Roll-over mouse to view the name of the workspace where the design is saved.
    • Type. The type of the design.
    • Rev. Current revision of the sequence.
    • Status. Status of the design in ENOVIA. One of the following is displayed for each design:
      • New. Displayed if the design does not exist in ENOVIA.
      • Modified. Displayed if the design is modified in AutoCAD since the design was last saved or opened.
      • Obsolete. Displayed if newer version of the design exists in ENOVIA database.
    • State. Current maturity level of the design in ENOVIA lifecycle.
    • Locked By. If the AutoCAD design being saved already exists in ENOVIA, and is locked by any user, then this column indicates the name of this user. You cannot select a design that is locked by another user.
    • Description. Description entered for the design by the user.

  2. You can enter the following details for saving the design.


    • . Click to select all the new and modified designs from the listed designs.
    • Save To. The ENOVIA workspace where the designs will be saved. For selecting a workspace, see Saving New Designs to ENOVIA
    • Comments. Enter comments for design you are saving. The comments are stored as an attribute in the design in ENOVIA.
    • View. Click and select a table defined by the Integration Administrator from the drop down list. The selected table is applied to the ENOVIA Save dialog box.

  3. Click Options to open the ENOVIA Save options dialog box:



    Select the options to be executed when you save selected designs to ENOVIA:

    • AutoName Series. Select a naming series from the drop down list. See Saving New Designs to ENOVIA. You cannot apply autonames for modified designs.
    • Background. Select to save designs without affecting AutoCAD operations. You can continue to use AutoCAD even when the save process is in progress.
    • Copy Attributes. Select to copy attributes from AutoCAD to ENOVIA. The attributes are transferred for only designs with Modified or New status. See About Saving in ENOVIA for more details.
    • Create Version. Select to create a new version of the design in ENOVIA. If not selected, the existing version of the design in ENOVIA is overwritten by the design being saved.
    • Retain Lock. Select to retain lock on the design. It is recommended that you lock your designs while saving to prevent other users from modifying your designs. If not selected, the design is saved to ENOVIA in the unlocked state.
    • Delete Selected Local Files. Select to delete the files of the design on your local disk upon saving.

      Note: Delete Selected Local Files option cannot be selected if Background option is selected.

    Click OK.

  4. Right-click the design and select:


    • Autoname. Click to assign a name automatically to the selected designs. See Saving New Designs to ENOVIA
    • Revise. Click to apply the target revision specified in the Save Options to all the selected designs. The revision number or sequence is displayed under the Target Revision column. For new designs, the first revision in the revision sequence is applied as the revision by default. For example, Rev A. See Saving New Designs to ENOVIA for more details.

      Note: If the value of the attribute IEF-AllowLeapFroggingInCheckin is set to FALSE in the user's GCO, then the user is not allowed to save an obsolete design to ENOVIA. The user is neither allowed to create a new version in the same revision stream nor allowed to create a new revision of the design.

    • Revise All. Click to revise new and modified designs from the ENOVIA Save dialog box which are revisionable, in a single go. See Saving New Designs to ENOVIA for more details.
    • Undo Revise. Use this option to revert the target revision added using the Revise option. This option is available only for nodes that were set the target revision using the revise option.

  5. Click Save when done.

The designs are saved in ENOVIA with the options selected for save applied.

Note: Attribute Synchronization does not happen for ACAD Drawings containing block attributes.
