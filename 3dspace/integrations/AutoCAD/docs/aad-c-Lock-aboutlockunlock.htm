About Locking and Unlocking

Locking a design in ENOVIA reserves the right to modify the design for the user.

Locking and Unlocking Designs

The Lock feature is used to lock a design in ENOVIA to ensure that no other user can overwrite the design with their changes. For example, if Part A is locked by User A, then another user, User B cannot modify Part A.

It is recommended to retain the lock for your designs while saving the designs to ENOVIA. Leaving a design unlocked in ENOVIA allows other users to modify the design and save their changes to ENOVIA.

Read Only Behavior for Family Objects

The following are the behavior changes while managing configurations in ENOVIA depending on the lock status of the files.


  • If family object is locked, then the files that are downloaded upon checkout, are writable on the disk.
  • If a configuration is unlocked which causes unlock of all the instances also, then the files are writable upon checkout.
  • If the instance objects of a configuration is locked, then upon checkout the files are made read-only on the disk.
