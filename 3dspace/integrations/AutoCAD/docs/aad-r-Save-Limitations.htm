Limitations in ENOVIA Save

This section lists some limitations during Save.

  • Incorrect Status Display

Incorrect Status Display

The following limitations cause ENOVIA to display incorrect status of the files in ENOVIA Save dialog box.

AutoCAD 2010 Limitation

AutoCAD 2010 does not recognize (xData) modifications done on AutoCAD 2000/LT 2000 Drawings. Hence, Collaborative Design for AutoCAD does not recognize drawings with xData modifications as Modified. So it is not possible to identify modified AutoCAD 2000/LT 2000 Drawings for subsequent ENOVIA Save operation in AutoCAD 2010.

AutoCAD Mechanical Limitation

This issue is specific for AutoCAD Mechanical users.

A drawing file is opened in AutoCAD Mechanical and saved to ENOVIA using ENOVIA Save option. When the user clicks File > Save or "Ctrl+S" after saving to ENOVIA, the file is considered as modified by AutoCAD even if no actual modifications are done in the drawing file.

When you try save the drawing again to ENOVIA, the Status column in ENOVIA Save dialog box displays the status for the drawing as Modified.

The Autodesk Support Request Number for this issue is 1261244.

Incorrect Modification Status of AutoCAD Drawing files due to Miscellaneous Modifications

Any operations like zoom, pan, layout switching and so on, marks the drawing file modified in AutoCAD. When you try to save the files after such operations, the ENOVIA-Save dialog box displays the status of these drawings as modified. There is no feasible way to identify geometry modifications and other miscellaneous modifications.

The Autodesk Support Request Number for this issue is 1259136
