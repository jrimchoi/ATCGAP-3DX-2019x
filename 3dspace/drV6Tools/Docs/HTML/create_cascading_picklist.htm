Create Cascading Picklist


Contents

In this walkthrough, we will setup the 3dxtools configuration required to create DRL DOCUMENT objects with predefined types and subtypes. Using this configuration user can create documents easily and with the combination of type and Subtypes, the policy, attributes and document templates are auto updated. You can use a standard ENOVIA operation trigger to configure create document process. This is basically used to configure the front end of the application for ease of use.

This package contains the complete configuration including reference files and test plans required to customize create document process.

When users create a new document this configuration will run through the customer specific activities and display the create form a with the user defined options. On selecting the value in one field will run the dependent configurations and it will reload other fields based on the combination and configuration.

•Select Type and Subtype instead of typing manually.
•Custom form for create document based on the combination of type and subtypes.
•On submit:
oApply a document template and route template
oSet default attributes
oAssign pre-defined prefix to the document name
oApply document lifecycle to be used in review and approval

 

This configuration will provide the flexibility to control document behavior using least number of inputs while creating the document by proving predefined values in the UI. This feature of 3dxtools will reduce the input time during the data creation by large extent. This will help in implementation of requirement changes related to document lifecycle, customer can easily setup the user-friendly environment to create the customized documents.

It will simplify the product configuration changes as lookup values are controlled in a simple layout file.

Overall implementation time and data creation will be reduced as most of the document properties will be automated in lookup activities.

Following are the 3dxtools functionalities covered in this walkthrough:

1.Create Folder object

The 3dxtools Folder is the place holder object for 3dxtools triggers. It is used to combine and display the configured activities for a standard ENOVIA operation in an expanded view.

 In this example, we will create a placeholder for ENOVIA Create operation.

2.Create Type Action Trigger object

The 3dxtools Type Action Trigger is used to connect an action event on a Type in ENOVIA.

In this example, we will connect Create event to the DRL DOCUMENTS within the Workspace folder and Create the simple list for create DRL DOCUMENTS cascade actions.

3.Create Cascade Activity object

This feature of 3dxtools provides user as simple classification in a system at simple clicks in the application. This will contain multiple level picklist options which will provide multiple options to the user based on the previous values selected during the process of create/modify.

This feature is used in ENOVIA to provide friendly environment where user can select the standard values, which will help in providing unique identifiers. This can be used to build the URLs of the application dynamically based on the selection when its redirected to ENOVIA web user interface.

In this example, we will be using this feature to create DRL DOCUMENTS object using the customized create form using Cascade options.

4.Create Cascade Selection List object

This feature of 3dxtools provides user to create list control in a cascade feature where user can pick from the list of predefined values. This will help users to create objects without the reference of any other document as the values will be predefined and it can be easily selected from the list available. This will help in reducing the time to type the values in field and helps in avoiding mistakes during the creation or modification of the values.

In this example, this feature is used to provide filed that lists the types and subtypes in creation of DRL DOCUMENTS. The Types are displayed as combo box in create from and based on type selection, the sub types are displayed in the form of combo box from which user can pick the values.

5.Create Sequence Number Activity object

This 3dxtools Sequence Number activity feature provides user to customize ENOVIA to update the sequence of the newly created/processed DRL Document object.  Basically, this feature is used to define/update the sequence of the object. For example, the sequence if the object can be 1 by default. If user wants to have it as A or any other sequence as default or on some action, it can be achieved using this feature of 3dxtools.

In this Example, the default sequence of DRL DOCUMENT object is updated on creation based on the type_DRL DOCUMENTS object and number generators.

6.Create Cascade Selection List Option objects.

This feature in drV6Tool provides the capability of defining the values for Cascade picklist under Cascade selection list. User can define any number of values using Selection list option, which will be populated under the Cascade Selection List object. These options mentioned will be populated in different forms (ex: combo box, chooser etc) based on the settings defined in Selection List object.

In this example, we have used this feature to populate the values of types and sub types in create page of DRL DOCUMENTS.

7.Create Cascade Selection Free Text

This feature provides user to have free text field in the customized form created by user in the ENOVIA environment. This field is basic text field like the text field available in ENOVIA, where user can define validations easily without much effort.

In this example, we have used this 3dxtools feature to provide Label, Description, Customer title etc. during the creation of DRL DOCUMENTS object.

8.Create Cascade Selection Search List

This 3dxtools feature is used to populate the values in one field based on the value defined in the previous field. When user selects some values in one field, this feature will search the corresponding values to populate in another field.

In this example, Sub Type field is populated with the values based on the value selected in Type field in Create DRL DOCUMENT form.

9.Create Copy Values Target Attribute

This feature allows administrators to manipulate the values on an object as part of any operation in ENOVIA. The activity makes it possible to carry out complex string manipulations between ENOVIA objects.

In this example, we will use name (manual entry) and Sub Type of document for the copy value attributes.

10.Create Copy Values Element Object Attribute

This feature provides the ability to calculate a single value. Several of these objects can be linked to a Copy Values Target Attribute to construct more complex values, using the Order Number attribute on the relationship to control the order in which the values are calculated and added together.

In this example, we will link Copy Values Element Object Attribute objects to document Name and Sub Type target attributes.

11.Create If Condition Object

This feature provides customized environment for the user to perform any operation during the creation of the DRL Document object based on the defined set of conditions. Basically, this is activity controls the dependent activity to perform its operation if the condition mentioned in this is passed.

In this example, based on the Type and subtype the name field will be displayed in UI.
