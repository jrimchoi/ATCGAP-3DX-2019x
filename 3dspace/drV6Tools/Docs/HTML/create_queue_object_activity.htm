Create Queue Object Activity


Contents

The 3dxtools Create Queue Object Activity enables the creation of queue objects which queue the current object for a server side processing job. The activity is used in conjunction with the 3dxprocessor Job Server, which picks up queue objects and processes them. This allows for batch operations or long running jobs to be moved onto the job server for processing. Queue objects can be monitored while they process, and provide a full, detailed output once processing is complete.  Examples of use are generating queue objects which:

•Generate a PDF view file using the View File Generator Processor
• Generate a large excel report using a combination of the Activity Processor and the Excel Report Generator Activity.

 

Configuration and Settings

To setup a 3dxtools Create Queue Object Activity:

1. Add a drV6tools Create Queue Object Activity object ➣

•Select the Dependant Activities tab on either a top level 3dxtools Triggers action object or another 3dxtools Activities object.
•Click on Actions.
•Select Create DRLTools Create Queue Object Activity...

oThis will open the DRLTools Create Queue Object Activity create form.
•Set the following values:

Processor

Defines the name of the Job Server Processor that should pick up and process the queue object.

Use Shadow Agent

TRUE

or

FALSE

TRUE (ticked) means the system will use the background system super user to create the queue object. This is typically needed as usually standard users do not have the required permissions to create a queue object.

FALSE (unticked) means the system will not use the background system super user to create the queue object.

Priority Order

Any integer value. Default value is 1

Defines the processing priority of the Queue Object created. System will add priority order value to all Queue Objects created using this activity. Jobserver picks the waiting queue objects and sort based on priority value and start processing.

If initially multiple queue objects are added in processing queue and processing started. Now if any new high priority queue object is being added to this queue then higher value will be picked in place of already available low priority objects in initial list.
