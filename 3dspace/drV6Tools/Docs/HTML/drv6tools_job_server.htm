3dxprocessor


Contents

The 3dxtools Job Server Service 3dxprocessor is a general multi-purpose server side job processing platform designed to handle any offline task required by an 3DEXPERIENCE ENOVIA implementation (e.g. PDF view file generation). The system runs as an unattended windows service with configuration and management handled through the ENOVIA standard UI including live feedback on current processing jobs. The 3dxprocessor has been designed as a effective extendable server side job processing platform running as a multi-threaded application, able to handle multiple jobs at the same time.

Each 3dxprocessor installation can handle multiple 3dxprocessor instances. Each of these instances create their own windows service that can connect to a 3DEXPERIENCE ENOVIA environment and run processing jobs. This allows for a single instance to connect to multiple 3DEXPERIENCE ENOVIA environment, which is particular useful in test and validation environments. The 3dxprocessor instances are created and managed using the 3dxprocessor Settings Manager application.

Each 3dxprocessor instance will create and mange it own windows service. You can view, start and stop the service either from inside the 3dxprocessor Settings Manager  or using the normal Services application.

Typically a 3dxprocessor will be installed locally to the MCS to handle any jobs that required a lot of database communication (for instance report generation jobs). 3dxprocessor installations should also be done on each FCS site. These FCS installation will be used to handle any jobs that manipulate and check in or out files (for instance view file generation ) and ensure that the file communication is kept locally to the FCS.

Each 3dxprocessor installation instance will be represented and managed within ENOVIA by its own Job Server Controller object. This object allows configuration of that particular installation instance and provides the live feedback of its current progress.
