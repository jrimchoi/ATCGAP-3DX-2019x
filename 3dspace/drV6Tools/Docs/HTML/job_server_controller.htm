3dxprocessor Job Server Controller


Contents

Each Job Server installation will be represented and managed within ENOVIA by its own Job Server Controller object. This object allows configuration of that particular installation and provides the live feedback of its current progress.

To access the Job Server Controller objects click the My Enovia button and then select 3dxtools Processor Settings (you need to be a member of the DRLTools Admin role to be able to see this menu).

The 3dxtools Processor Settings menu will bring up a powerview showing all Job Server Controller objects in the top channel and any Waiting or Failed Queue Objects in the bottom channel.

To create a new Job Server Controller object click the Create Job Server Controller button:

oThis will open the DRLTools Job Server Controller create form.

It is requested as part of the Job Server installation process and is used by the installation to connect the physical job server installation with the Job Server Controller object that will control its configuration and execution.

Source

{Any suitable value}

Stop Processing

TRUE/FALSE

This value provides a mechanism to stop the Job Server installation from processing any more queue objects. Setting this value to TRUE will cause the Job Server to shutdown to a minimum level. Any queue object that has started processing will be allowed to complete, but no further queue objects will be started. The Job Server itself remains active but its only function is to monitor this value. When set back to FALSE the system will start processing queue objects again.

Description

{Any suitable value}

 

•The following values are set by the Job Server installation service and should not be changed.

Read-only Attribute Name

Value/Notes

Process Control Status

This is to show if a 3dxprocesor is currently active and running,single colored icon (Green cube or Grey cube) to show if the processor it is controlling is active or not.

This should be based of the modified date of the Processor System Controller object.

If it was modified in the last minute, it should be Green else Grey.

Process Object Status

This is to show the health of the currently processing queue objects. The ProgramHTML output column will show a list of coloured icons,one for each linked Processing Queue Object and it will be coloured based on the modified date of the Processor System Queue Object.

The icon will also be a hyperclick that if clicked will take the user directly to the queue object that it refers to.

•If it was modified in the last minute, it should be Green
•If it was modified in the last 5 minutes, it should be Amber/Orange.
•If it was modified in the last 10 minutes, it should be Amber/Orange.

Below shows the Job Server Controller power view which is made up on three main components:

•The top left channel shows the properties of the Job Server Controller object and by refreshing this windows you will see the Summary and Process Details attributes being updated with live information from the job server installation.
•The top right channel shows information on the Queue Objects related to the Job Server Controller with a separate command tab for each of the queue object states. The default one displayed shows the Queue Objects currently being processed by the job server.
•The bottom channel shows the command tabs for managing the Job Server Triggers objects which controls how and when the system gets the Queue Objects to process and the and Job Server Processors objects that holds the configuration on how these queue objects are processed.
