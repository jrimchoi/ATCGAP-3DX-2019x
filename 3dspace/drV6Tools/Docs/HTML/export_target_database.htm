Export Target Database


Contents

The 3dxtools Export Target Directory object exports the metadata selected as part of the parent Export Activity to any database that supports OLEDB. The XML metadata generated is passed into a defined large string field in a defined data table within the database. Other values can then be set using Copy Values Activity by attaching Copy Values Target Attribute and setting the Target Value attribute to the name of the database column you wish to set. This export target is typically used when you wish to use a database as the intermediary between the ENOVIA export and the import into a different 3rd party system. This allows you to use the intermediary database as a buffer between the two systems.

The system will by default use the 3dxtools XML export format to output the metadata of the selected exported objects. The system provides the ability to parse the XML file through an XML translation process by enabling the administrators to define a standard XML EXtensible Stylesheet Language Transformation (XSLT) setting. Details on XSLT can be found here: XSLT Tutorial.

 

To create an Export Target Database object:

•Select the Export Targets  tab on the Export Activity object
•Click Actions.
•Select Create Export Target Database...
oThis will open the DRLTools Export Target Database create form.
•Set the following values:

Database Connection String

{A valid OLEDB connection string}

This defines the OLEDB connection string to use to connect to the database to export to. It is a requirement for this to work that the MCS or Job Server machine that is running this activity has the ability to connect to the database using this string.

Database User Name

{The OLEDB database user name}

This defines the user name to use to connect to the OLEDB database.

Database Password

{The OLEDB database password}

This defines the password to use to connect to the OLEDB database.

Database Table Name

{The database table name}

This defines the table within the OLEDB database that should be used to write the data to.

Main Data Column Name

{The database table column name}

This defines the column within the database table (defined Database Table Name) that the main XML data should be exported to.
