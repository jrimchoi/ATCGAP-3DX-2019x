Get Actions


Contents

The 3dxtools Get Actions provides the ability to execute the actions on related object of context object. If the child action returns failure then the get action will not pass and further child actions will not be executed.

Get actions will not block the current operation. But if any dependant check action returns failure then current operation will be blocked and child action’s validation message will be displayed to the user

 

Configuration and Settings

To setup a 3dxtools Get Actions:

1. Setting Configuration ➣

•Add high level business logic in below format to any Action Activity object Settings attribute.

<actions>

 <get object="mainObject">

         <actions>

                 <actionActivity name="CheckLockStatusOnMain"/>

         </actions>

 </get>

 <get object="version">

         <actions/>

 </get>

 <get object="latestVersion">

         <actions/>

 </get>

 <get object="previousMinor">

         <actions/>

 </get>        

</actions>

 

2. Object Setting Details  ➣

Setting

Description

mainObject

Defined child actions under this get action will be executed on main object of document.

version

Defined child actions under this get action will be executed on version object of document.

latestVersion

Defined child actions under this get action will be executed on version object of document.

previousMinor

Defined child actions under this get action will be executed on previous minor of object.

previousMinor will be calculated using previousminor mql selectable on context object and if it exists then previousMinor object will be retrieved using below query:

temp query bus TYPE NAME PREVIOUSMINOR select id;

actions

child actions configuration

 

•For child actions, add any Action Activity you wish to run from this event.
