Type Action Trigger


Contents

The 3dxtools Type Action Trigger is used to connect an action event on a Type in ENOVIA with a series of 3dxtools Activities to execute when the event fires. Some examples of the operations you can perform using these triggers are:

•Use the New From Template Activity when adding an object to give it a template file.
•Use the Create Queue Object Activity when checking in a file to create a queue object that generates a view file.
•Use the Sequence Number Activity when creating an object to assign a unique sequence number as the name.

Configuration and Settings

To setup a 3dxtools Type Action Trigger:

1. Add the Event Trigger to the Type in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Type you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the type you wish to use by querying like  below,

  MQL>list type ${TypeName};

  MQL will list the type name.

•On the Triggers for that type find the ENOVIA trigger you wish to run the 3dxtools Activities on.
•To find the triggers which are already existing on type write the below query,

 MQL>print type ${TypeName} select trigger;

       This will print already existing triggers for the mentioned type including both inherited triggers(inherited from parent type) and immediate triggers(added on this type)

       Example: Print type "DRLTools Activities" select trigger;

   

    The MQL result will list the triggers which are already added

    trigger = CreateAction:emxTriggerManager(TypeDRToolsCreateActionTrigger)

    here,the trigger event is Create and trigger type is Action (for more details on events and trigger type refer below table).

    for the CreateAction already a trigger named TypeDRToolsCreateActionTrigger has added.

MQL>print type ${TypeName} select immediatetrigger;

  This will print already existing triggers for the mentioned type which are directly added on the type.

•To add new trigger to the type
•Edit the trigger.
•Set the following values:

Action

For details on emxTriggerManager please see the standard ENOVIA documentation.

Input

Append to any existing value

TypeAllDRToolsV6RunActionTrigger

or

TypeAllDRToolsV6RunActionTrigger_Deferred

TypeAllDRToolsV6RunActionTrigger runs the dependant 3dxtools Activities immediately.

TypeAllDRToolsV6RunActionTrigger_Deferred waits until the current transaction scope has completed before running the dependant 3dxtools Activities.

For Create event triggers in ENOVIA it is recommended to use TypeAllDRToolsV6RunActionTrigger_Deferred. This is because if you use the immediate trigger all you will have is the object with its type, name and revision set when the trigger fires. This typically isn't enough information to usefully run the 3dxtools Activities. Using the deferred trigger allows the current transaction scope to complete (which in this case is adding the object and setting all the initial values) giving you a fully created and defined business object to use.

 

Trigger_Type

Name of the Trigger Type :Action

List of Events: Check,Action,Override

Check: It performs an action before an event occurs.

Action: It performs an action by replacing an event.

Override:It performs an action after an event has occurred.

Event_Name

Name of the Event:Revise

Modify,Create,Revise,checkin.etc..please see the standard ENOVIA documentation for more events

to add new input to the event(Create) which already exits for the trigger type(Action),

Syntax:

MQL>mod type ${Type_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Example:

MQL>mod type "DRLTools Activities" add trigger Create Action emxTriggerManager input TypeAllDRToolsV6RunActionTrigger;

    Confirm the trigger added  from the below query:

    MQL>print type "DRLTools Activities" select triggers;

   

•Remove trigger:

To remove trigger from the type mention the event and trigger type, it removes all the trigger which got added

     MQL>mod type "DRLTools Activities" remove trigger Create Action;

   

    The specified trigger type is removed from the listed event except the trigger which got inherited from parent.

•Confirm the trigger added  from the below query:

   MQL>print type ${TypeName} select triggers;
