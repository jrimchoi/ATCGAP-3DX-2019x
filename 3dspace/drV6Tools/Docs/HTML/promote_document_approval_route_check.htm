Promote Document Approval Route Check


Contents

There should be a validation to check document attributes before promoting.

For example, If Document Subtype and Approval Route attributes are empty then there should to an approval route connected to the document.

 

1. Business Object and Execution  ➣

Name

Description

Promote_Engineering Document_Formal Release_WIP

and Promote_Engineering Document_Formal Release_Review

Check triggers on WIP and Review state of Formal Release policy.. Details at Validation Objects

Action Activity

DRLTools Action Activity_CheckCustomerNameAndAddressAreNotBlank_1. Details at Validation Objects

Execution

When document is being promoted from ‘WIP to Review’ or ‘Review to Approved’ then system will run if state action to check WIP or Review state with dependant if condition for Document Subtype and Approval Route attributes. If both attributes are empty then there will be no automated approval route for document lifecycle hence you have to create the approval route.

First check connection action will be executed to check approval route template connections. If document has related route templates then system will again execute check connections for Approval Route with Promote Connected Object action.

It document does not satisfy the above criteria then promote operation will be blocked and error message displayed to the user.
