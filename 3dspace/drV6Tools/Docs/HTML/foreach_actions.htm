ForEach Actions


Contents

The 3dxtools ForEach Actions provides the ability to test related objects of defined type that meet certain criteria. If the related object isn’t of the defined type then the further child actions will not be executed.

ForEach action will not block the current operation. But if any dependant check action returns failure then current operation will be blocked and child action’s validation message will be displayed to the user.

 

Configuration and Settings

To setup a 3dxtools ForEach Actions:

1. Setting Configuration ➣

•Add high level business logic in below format to any Action Activity object Settings attribute.

<forEach>

 <name>forEachLinkedDrawing</name>

 <query>expand bus ${id} rel 'Drawing Source' select bus id</query>

 <routeContent>false</routeContent>

 <ofTypes>

         <type>SW Drawing</type>

 </ofTypes>

 <actions>

         <checkState>

                 <name>CheckFrozenState</name>                                                            

                 <failAction>FailAndContinue</failAction>

                 <failMessage>${type} ${name} ${revision} needs to be at least frozen</failMessage>

                 <states>

                         <states onOrPast="true">FROZEN</states>

                 </states>

                 <actions/>

         </checkState>

 </actions>

</forEach>

 

•More uses in Validation Objects

2. Setting Details  ➣

Setting

Description

name

Any meaningful value. It is recommended to start with “forEach”.

ofTypes

Related object type to be checked. If the object is any of the configured types then system will pass the check and continue executing child actions. You can define exact type or parent type as system will always check for kindOf[TYPE].

query

Define the logic to get related object ids. If it is not the expand query then you need to define the objectIdIndex parameter to get id from result. For Example

<query objectIdIndex=”3”> temp query bus * "3dxtools Validation System*" * select id dump |</query>

If both query and pattern defined, then system will prefer the query setting

If it is expand query and multiple selectable used on relationship and object then define objectIdIndex and relIdIndex as given.

routeContent

It is an optional setting with default value as false. It will be applicable to route only. You can either define the query to get route content or set routeContent to true. If both query and routeContent defined then query setting will be ignored.

If the context object is of route type then Object Route relationship will be used to fetch the content else emxTableRowId parameter in requestParamMap will be used to get content ids.

emxTableRowId will be “,” separated list of object ids.

actions

child actions configuration

 

•For child actions, add any Action Activity you wish to run from this event.
