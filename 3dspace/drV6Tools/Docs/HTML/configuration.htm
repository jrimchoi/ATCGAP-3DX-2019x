Overview

3dxtools is configured by objects in the ENOVIA database, as represented in the diagram below. For an Activity  to run, there must be a trigger configured in Business for the type, attribute, relationship or policy and operation required. The trigger uses emxTriggerManager to pass an appropriate parameter to the 3dxtools program object (drToolsRunToolsJPO). When the trigger event occurs, the JPO searches ENOVIA for a Trigger object with the correct name. Once found, it traverses the relationships from the trigger object to find any top-level activity objects and their dependent activity-specific objects.

 

 

Configuration objects within ENOVIA are accessed from the My Enovia menu, under the 3dxtools Admin menu group. Selecting 3dxtools Configuration brings up the 3dxtools Configuration page.

Access to these menus are controlled using roles. Only users with the DRLTools Admin role will be able to see these menus.
