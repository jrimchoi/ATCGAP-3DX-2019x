View File Generator Inventor Settings


Contents

The 3dxtools View File Generator Processor for Inventor is designed to generate PDF view files for Inventor Drawing files checked into the ENOVIA store. As part of the process the system will open the file in its native Inventor application, carry out parameter mappings to the drawing and create a PDF.

 

Configuration and Settings

1. Add a View File Generator Inventor Settings object ➣

To add Job Server View File Generator Inventor Settings object:

•Select the View File Generator Settings tab on the View File Generator Processor object.
•Click Actions.
•Select Create Processor View File Generator Base Setting...
•Select the Type Selector
•Select Create Processor View File Generator Inventor Setting...

oThis will open the Processor View File Generator Inventor Settings create form.
•Set the following values:

MCADInteg-CADToMxAttribMapping

{Inventor Drawing to ENOVIA mappings}

This defines the mappings to perform from the Inventor Drawing to ENOVIA.

Each line is a mapping with the syntax for each being.

drawing,{Property name}|{Type},{Basic or Attribute}

i.e.

Mapping Title Inventor Drawing iproperty to the ENOVIA description attribute

drawing,$$Title$$|INV Drawing,$$description$$

Note: All properties within the Inventor Summary Information tab should be referenced using the $$ prefix and $$ suffix. e.g. Title property = $$Titile$$ and comments property would be $$Comments$$. All other custom iproperties can be referenced with just their name.

MCADInteg-MxToCADAttribMapping

{ENOVIA to Inventor Drawing mappings}

This defines the mappings to perform from ENOVIA to the Inventor Drawing.

Each line is a mapping with the syntax for each being.

{Type/Parent Type},{Basic, Attribute or Macro Values}|drawing,{Property name}

i.e.

Mapping ENOVIA revision to a iproperty called MXRevision

INV Drawing,$$revision$$|drawing,MXRevision

Note: All properties within the Inventor Summary Information tab should be referenced using the $$ prefix and $$ suffix. e.g. Title property = $$Titile$$ and comments property would be $$Comments$$. All other custom iproperties can be referenced with just their name.

View File Watermark

This allows you to have "Uncontrolled Document" stamps mapped to the drawing that will be set and shown on the source drawing (which is uncontrolled and can be changed by the user) and not shown on the PDF drawing (making this the controlled document).

2.Prepare the Inventor installation ➣

•Install and Activate the Inventor Application

Inventor must be installed and activated on the machine that will run the Job Server. To confirm that the installation has worked correctly you must log in as the Windows user who will run the Job Server windows service (this user must be a local administrator) and run the Inventor applications. Ensuring that the application run properly under that user without any dialog or popup windows opening. You should disable any add-ins (even any ENOVIA integrations add-ins) that are installed that may open a dialog window. As this system is designed to run unattended any dialog windows that open within the Inventor application whist the PDF process is running will cause the system to hang.
