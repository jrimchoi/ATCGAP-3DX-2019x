Activity Processor


Contents

The 3dxprocessor Job Server Activity Processor is designed to run any 3dxtools Activities as a batch operation. This processor is useful for running any 3dxtools Activities that require significant processing or resources that should not be executed on the MCS server. It can also be used for running operations on objects that are not triggered through user events within ENOVIA (i.e. triggering an updating objects matching search criteria after a certain amount of time has passed).

 

Configuration and Settings

1. Add a 3dxprocessor Job Server Activity Processor object ➣

To add Job Server Activity Processor:

•Select the Processors tab on the Job Server Controller
•Click Actions.
•Select Create Job Server Processor...
•Select the Type Selector
•Select the Type Job Server Activity Processor

oThis will open the Job Server Activity Processor create form.
•Set the following values:

This value is used to define the search to use to find the objects to run the dependant activities on. Before it is used the system will check for a Copy Values Target Attribute with a target attribute value of MQL_QUERY. If it finds a target value with a matching name it will use that value instead. If no target attribute is found then this value is used. This value accepts any valid Macro Value.

It does not need to be defined if your running the activities on the object you created the queue object on (I.e. on the promotion of an object you create a queue object to initiate a long running report to be generated). Also if your depentant activities do not require a business object then this value is not required.

Add Result Files To Referenced Object

TRUE or FALSE

This value is only used if the reference object in the queue object is set and defines if any file created as a result of the dependent actions should be saved and checked in to the reference object.

If set to TRUE (and if the reference object is set) then the files will be checked into the reference object.

If Set to FALSE then no action will be taken.

Create New Object For Result Files

TRUE or FALSE

This defines if any file created as a result of the dependent actions should be saved and checked in to a new Derived Output object. If set to TRUE the system will create a new Derived Output object and save and check in the files to it.

Derived Output Policy Symbolic Name

{Policy}

If the value Create New Object For Result Files is set to TRUE then you must set this value which defines the name of the Policy to use for the new object.

Derived Output Type Symbolic Name

{Type Name}

If the value Create New Object For Result Files is set to TRUE then you must set this value which defines the name of the Type to use for the new object.

Derived Output Object Uses Common Document Model

TRUE or FALSE

If the value Create New Object For Result Files is set to TRUE then you must set this value which defines if the Derived Output object uses the common document model. If set to TRUE the system will save the files to the object using the common document model requirements. If set to FALSE the files will be just checked into the business object.

Derived Output Relationship Name

{Relationship Name}

This value is only used if the reference object in the queue object is set and the value Create New Object For Result Files is set to TRUE. It defines the relationship name to use to connect the reference object and created Derived Output object together with. The system will only try to create a relationship if this value is set. By default no relationship is created.

Derived Output Object Generator Name

{Object Generator Name}

If the value Create New Object For Result Files is set to TRUE then you must set this value which defines the name of the eService Object Generator object to use for the name for the new Derived Output object.

View File Format

{Valid Enovia File Format}

If either of the values Add Result Files To Referenced Object or Create New Object For Result Files are set to TRUE then you must set this value. It defines the ENOVIA format to use for the files to be checked into the objects.

Subject

{Email Subject}

This value is used to define the subject text of the message sent to the reference user once the dependent activities have completed processing. A message will only be sent if this attribute contains a value. If left blank then no message will be sent. This attribute can use Macro Values.

Message

{Email Message}

This value is used to define the message text sent to the reference user once the dependent activities have completed processing. It is only used if the Subject is defined. This attribute can use Macro Values.

Real Time Feedback

TRUE or FALSE

If this value is set to TRUE then the Job Server will continuously update the Queue Object in the database with the progress feedback. If set to FALSE the process details will only be saved the database once the process has completed.

2. Add Dependent Activities to Run ➣

The dependent activities define the actions to perform on the object(s) passed in.

•Select the Dependent Activities tab on the If Condition Activity object.
•Click Actions.
•Select the 3dxtools Activities required.
