Configurable Drag and Drop Cascade Action


Contents

3dxtools provides a simple mechanism to configure the Enovia drag and drop with document classification using 3dxtools Cascade. It gives the ability to create an object with document classification by dropping a file onto the drop zone and to check in documents to an object using 3dxtools Cascade.The feature can be turned on / off based on the configuration setting which is created against the type Menu object in Business.

 

For example, below shows settings configured to show drag and drop with document classification using 3dxtools Cascade.

Configuration and Settings

To setup the 3dxtools Configurable Drag and Drop with Cascade Action:

1. Modify the OOTB ENOVIA Menu Object to Add Settings ➣

•On the type_{Business type}  menu object, select and edit the settings.
•Add the following settings to configure  the cascade activity with drag and drop.

Setting Name

Value

Notes

dr cascade activity

{true or false}

Turns on the object creation using 3dxtools Cascade . If set to true it requires the dr cascade activity name . If blank or not set the default of false is used.

dr cascade activity name

{a suitable description}

This defines the name of the 3dxtools Cascade Activity that should be run when the file is dragged.

dr cascade target location

{slidein or popup}

This defines the target location for cascade form to appear.

By default it is slidein where the cascade form appears in a slidein frame (slides in along the right side of the window).

popup-- cascade form  appears in new window.

dr run cascade for file extensions

{List of file extensions}

e.g.

docx,doc,xls,ppt,xlsx,pptx

Defines the file extensions that Cascade activity should run for.  File extensions should be entered without the preceding ".", and separated with commas.

dr cascade activity conditions

{a list of If Condition Activity

objects to check delimited by '|'}

This defines the list of If Condition Activity object names to check to see if the 3dxtools command activity should be displayed. It is possible to define multiple names of If Condition Activity objects, if any of them evaluate to TRUE the system will display the command icon. If all of them evaluate to FALSE the icon will not be displayed.

2. Include the script tag in emxNavigator.jsp ➣

Include the script tag   <script src='../3dxtools/draganddrop/js/drExtendedPageHeaderFreezePaneValidation.js'></script>

in emxNavigator.jsp which is can be found in enovia\common directory after the script tag listed below.

<script src='scripts/emxExtendedPageHeaderFreezePaneValidation.js'></script>

3.Rename drlTools_emxExtendedHeader Program ➣

If you are using Enovia version 2015x or higher , search  the program in Business called 'emxExtendedHeader' and clone it to take a backup.

Then search and rename the program 'drlTools_emxExtendedHeader' to  'emxExtendedHeader' in order for the drag and drop functionality to work.
