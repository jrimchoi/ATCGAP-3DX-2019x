Description


Contents

 In this walk-through, we will setup the 3dxtools configuration required to define different excel and pdf report layouts. This package contains the complete configuration including reference files to configure the workspace content and Issues status report.

This configuration will provide the flexibility for the customer specific report layouts. This package enables the user to extract real time information from the database and present the data within Microsoft Excel or pdf format. The report layout is built up from an XML configuration file.

For example, status reports for meetings can be generated just prior to, or during a review of a specific subject.

Following are the 3dxtools functionalities covered in this walkthrough:

1.Create Action commands in ENOVIA

In this walkthrough package, we will create ENOVIA command\menu schemas to trigger the report activities in UI.

2.Create Folder object

The 3dxtools Folder is the place holder object for 3dxtools triggers. It is used to combine and display the configured activities for a standard ENOVIA operation in an expanded view.

 In this example, we will create a placeholder for Action Triggers required to configure reports.

3.Create Type Action Trigger object

The 3dxtools Type Action Trigger is used to connect an action event on a Type in ENOVIA. We can create placeholder Action trigger for activities to be configured in Command Triggers.

In this example, we will create placeholders for excel and pdf report activities.

4.Create Excel Report Activity

The 3dxtools Excel Report Activity feature allows a user to create a detailed report, showing all or some attributes from objects and any number of relationships those objects have. The report is built up from an XML configuration file.

In this example, using this feature a report is generated to provide the user with a summary of all 'Open' Issues and documents details in any workspace within ENOVIA. A graphical view will be generated for document status and types.

5.Create Cascade Activity object

This feature is used in ENOVIA to provide friendly environment where user can select the standard values, which will help in providing unique identifiers. This can be used to build the URLs of the application dynamically based on the selection when its redirected to ENOVIA web user interface.

In this example, we will use this feature to build dynamic workspace report URL and forward for processing based on Name filter.

 

6.Create Cascade Selection Free Text

The 3dxtools Cascade Selection Free Text feature allows you to define a text box control in a Cascade, either to get input from the user or to display text. It is generally used for editable text, but can also be made read-only. It can display multiple lines.

In this example, we will use this to configure Name filter for workspace content report.

7.Create PDF Report Activity object

This 3dxtools PDF Report Activity feature allows for a user to create a detailed report, showing all or some attributes on an object and any number of relationships those objects have. The report is built up from an XML configuration file.

In this example, we will use this 3dxtools feature to provide company specific header and footer (copy right information) for any PDF report.

 

8.Create Queue Object Activity object

The 3dxtools Create Queue Object Activity enables the creation of queue objects which queue the current object for a server side processing job. The activity is used in conjunction with the 3dxprocessor Job Server, which picks up queue objects and processes them. This allows for batch operations or long running jobs to be moved onto the job server for processing. Queue objects can be monitored while they process, and provide a full, detailed output once processing is complete.

In this example, we will configure queue object for workspace content report.
