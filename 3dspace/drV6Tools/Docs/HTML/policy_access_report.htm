Policy Access Report


Contents

3dxtools Policy Access Report provides ability to generate text or log file for the user access defined on policy admin object. The content of this file can be viewed in macro enabled policy access display template (PolicyAccessDisplayTemplate.xlsx) file.

 

3dxtools Policy Access Report requires steps detailed below:

 

exec prog drExportUtility -method printPolicyAccessForExcel {POLICYNAME} {OUTPUTPATH} {ACCESSVALUE} {SEPARATOR}

 

Export Utility Parameters

•POLICYNAME- This needs to be a policy name in database.
•OUTPUTPATH – This is the path where policy access report log file will be created. User  should enter the path including file name.
•ACCESSVALUE – This is not the mandatory input. By default all access will be exported. User can export filtered access by passing comma separated list of database access name i.e. read,modify,delete etc
•SEPARATOR – This is not the mandatory input. By default '--------------------------------------------' separator will be used between state user accesses. User can pass any other separator string.

 

Create Report

•Run MQL as administrator

MQL commands to be used to export policy access

Example 1. Export all access with default separator

set context user creator;

exec prog drExportUtility -method printPolicyAccessForExcel "Design TEAM Definition" "c:/temp/drlPrintPolicyAccess_Design TEAM Definition.log";

 

Example 2. Export selected access with custom separator

set context user creator;

exec prog drExportUtility -method printPolicyAccessForExcel "Design TEAM Definition" "c:/temp/drlPrintPolicyAccess_Design TEAM Definition.log" "read,modify,show" "*********************";
