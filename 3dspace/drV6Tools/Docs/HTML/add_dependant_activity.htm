Add Dependent Activity


Contents

The 3dxtools Add Dependent Activity ,functionality is available on content page of Change Action ,it allows owner of CA to add dependents.

This Action Activity will run to get the list of objects added to the Proposed Change, then looping each object to get dependent objects and it will be connected to Change Action.

Configuration and Settings

To setup a 3dxtools Add Dependent Activity:

1. Setting Configuration ➣

•Add high level business logic in below format to any Action Activity object Settings attribute.

<setting>

 <actions>

         <ChangeActionAddDependantObjectsAction>

            <AddForTypes>

                         <AddForType>Part</AddForType>

              </AddForTypes>

              <CheckRelationships>

                         <CheckRelationship type="Part" relationshipPattern="Part Specification"              

                      typePattern="ProE*,Inventor,DOCUMENTS" direction="from" includeStates="IN_WORK,Review,Preliminary" excludeStates="APPROVED"/>

              </CheckRelationships>

        </ChangeActionAddDependantObjectsAction>

  </actions>

</setting>

 

2. Setting Details  ➣

Action

Description

AddForType

Type to be checked,when activity runs it will fetch the list of proposed changes ,then the type of proposed change will be checked with type in AddForType,if it  exists then activity will continue or else it will be terminated.

CheckRelationship

If type matches,CheckRelationship will be looped to get related objects.

type

Type it will be same as type which is mentioned  in AddForType.

relationshipPattern

Relationship name it can be multiple with comma separator ,to do get the dependents to Change action .

typePattern

Type name it can be multiple with comma separator to get the dependents.

direction

To check connection direction it can be  "from" or  "to".

includeStates

To get dependent objects which are having current state as mentioned in IncludeStates.

excludeStates

To get dependent objects by excluding by there current state.
