View File Generator AutoCAD Settings


Contents

The 3dxtools View File Generator Processor for AutoCAD is designed to generate PDF view files for AutoCAD Drawing files checked into the ENOVIA store. As part of the process the system will open the file in its native AutoCAD application, carry out parameter mappings to the drawing and create a PDF.

 

Configuration and Settings

•Select the Type Selector
•Select the Type Processor View File Generator AutoCAD Setting

oThis will open the Processor View File Generator AutoCAD Settings create form.

MCADInteg-CADToMxAttribMapping

{AutoCAD Drawing to ENOVIA mappings}

This defines the mappings to perform from the AutoCAD Drawing to ENOVIA.

Each line is a mapping with the syntax for each being.

drawing,{Property name}|{Type},{Basic or Attribute}

i.e.

Mapping MXDescription AutoCAD Drawing property to the ENOVIA description

autocad,MXDescription|Drawing,description

MCADInteg-MxToCADAttribMapping

{ENOVIA to AutoCAD Drawing mappings}

This defines the mappings to perform from ENOVIA to the AutoCAD Drawing.

Each line is a mapping with the syntax for each being.

{Type/Parent Type},{Basic, Attribute or Macro Values}|drawing,{Property name}

i.e.

Mapping revision to a AutoCAD attribute reference called MXRevision

Drawing,revision|autocad,MXRevision

View File Watermark

{List of attribute values to clear before mapping values}

This defines a list of attribute values that should be set to blank before creating the PDF. After the PDF is created the values are then reset to their original value. This allows you to have "Uncontrolled Document" stamps mapped to the drawing that will be set and shown on the source drawing (which is uncontrolled and can be changed by the user) and not shown on the PDF drawing (making this the controlled document).

CAD Type

{Derived Output CAD Type Value}

This value lets you set the Derived Output's CAD Type value. This is required for objects that may have multiple Derived Output formats as a way of identifying the specific one required.

Get Process Model Only

{TRUE or FALSE}

If set to TRUE the process will only plot the Model space and not the layout spaces. If set to FALSE then it will plot all the layout spaces.

Use Default Page Size

{TRUE or FALSE}

This defines if a default printer page size should be used for the creation of the PDF. If set to TRUE then you must define the setting value Default Page Size. If set to FALSE then you must define the setting value and every AutoCAD drawing object MUST contain the page size as an attribute value.

Default Page Size

{Default printer page size name}

This is the Printer page size name that should be used by default if the Use Default Page Size is set to TRUE.

Drawing Size Attribute Name

{Attribute containing page size}

This setting allows you to vary the page size used to create the PDF based on an attribute value on the business object in ENOVIA. If set to a value then this will override any value in the Default Page Size and be used as the print size. The values used on the business object must be either a valid printer page size or have an entry in the setting Printer Page Size Name By Page Size.

Printer Page Size Name By Page Size

{List of mapping with each line consisting of:

   {page size attribute value}={printer page size name}

}

This value provides the ability to map from the business object attribute value to the printer page size name and is useful with the display values on the business object do not exactly match the required printer page size names.

Page Dimensions To Size Mapping

{List of page dimensions to size mappings

 with each line consisting of:

 

   {width}*{height}={printer page size name}

 

   or

 

   {width from}-{width to}*{height from}-{height to}={printer page size name}

}

This value provides the ability to map from the current layout's width and height in millimeters to the printer page size to use. It allows the system to dynamically select the best page size to use based on the size of the layout. Values should be rounded to the nearest mm.

i.e.    279 * 216 = Letter

        297 * 210 = A4

This setting also allows you to define a range of values to allow you to select the best fitting page size based on a custom layout size.

i.e.    100-203 * 100-127 = Junior Legal

        204-216 * 128-140 = Half Letter

        217-279 * 141-216 = Letter

        280-356 * 141-216 = Legal

The system will run down the list of values and use the first matching value it finds.

Plot Orientation By Page Sizes

{List of mapping with each line consisting of:

   {printer page size name}={plot orientation}

}

The setting value defines the plot orientation to use based of the defined page size. This means you can have Landscape A2 and A3 plots and Portrait A4 plots. The orientation is defined as 0, 90, 180 or 270 and identifies by how many degrees the plot should be rotated. If left blank for a sheet the default orientation of 0 degrees will be used.

The syntax for each line should be:

{printer page size name}={0, 90, 180, or 270}

Fit To Page

{TRUE or FALSE}

If set to TRUE the plot will be scaled to fit to the page size

Use Drawing Style Sheet

{TRUE or FALSE}

This setting value defines if the processor should use a used defined style sheet. If set to TRUE the processor will use the drawing style sheet defined in Drawing Style Sheet Name. If set to FALSE it will use the default style sheet for the drawing.

Drawing Style Sheet Name

{style sheet ctb file name}

The ctb style sheet file name to use. This file must be in the plot styles path of ACAD in order for ACAD to locate it

Do Not Reset Drawing Style Sheet

{TRUE or FALSE}

This setting value defines if the processor should use not reset the drawing style sheet back to the original if being set to a specific drawing style sheet within the processor system. If Use Drawing Style Sheet = TRUE and this value is set to TRUE the processor system will leave the drawing style sheet to the    one defined in Drawing Style Sheet Name. If set to FALSE the drawing will be reset back to what it was originally

Use Block Names For Mapping

{TRUE or FALSE}

If this value is set to TRUE then the processor will also use the AutoCAD drawing block name concatenated (using the character defined in the setting Block Name Attribute Name Separator) with the attribute reference name to check the MCADInteg-CADToMxAttribMapping and MCADInteg-MxToCADAttribMapping for any attributes to map. If set to FALSE the system will only use the attribute reference name for the mapping

Block Name Attribute Name Separator

{separator character}

Character to use to concatenate the block name with the attribute reference name before checking the properties mappings to see if it should be mapped. Required if the value Use Block Names For Mapping is set to TRUE.

Check Block Names Before Mapping

{TRUE or FALSE}

This setting defines if the system should check the block name before checking to see if any attributes within it require mapping too for the ENOVIA to AutoCAD parameter mapping. If set to TRUE the setting Block Names Containing Mappings must be defined.

Block Names Containing Mappings

{List of Block Names containing elements to map to}

List of Block Names that contain text attributes that need to be mapped to with the ENOVIA AutoCAD properties mapping.

Enable Rotate PDF

{TRUE or FALSE}

This provides a utility to rotate the full PDF once it has been produced by the PDF printer and provides another opportunity to solve orientation issues that might be introduced by the AutoCAD printing process. If set to TRUE the setting Rotate PDF By must also be defined.

Rotate PDF By

{List of mapping with each line consisting of:

   {printer page size name}={PDF rotation}

}

The setting value defines the PDF rotation to use based of the defined page size.  The rotation is defined as 0, 90, 180 or 270 and identifies by how many degrees the plot should be rotated. If left blank then the PDF will not be rotated.

Debug Active

This defines if the system should activate the debug mechanism. This will save to file the inputs that are sent to the .NET component by the Job Server and also save its output to file.

Debug Output Directory

{Debug Output Directory Path}

This defines the debug output directory.

2.Prepare the AutoCAD installation ➣

•Install and Activate the AutoCAD Application

AutoCAD must be installed and activated on the machine that will run the Job Server. To confirm that the installation has worked correctly you must log in as the Windows user who will run the Job Server windows service (this user must be a local administrator) and run the AutoCAD applications. Ensuring that the application run properly under that user without any dialog or popup windows opening. You should disable any add-ins (even any ENOVIA integrations add-ins) that are installed that may open a dialog window. As this system is designed to run unattended any dialog windows that open within the AutoCAD application whist the PDF process is running will cause the system to hang.
