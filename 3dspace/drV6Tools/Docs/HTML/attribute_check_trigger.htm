Attribute Check Trigger


Contents

The 3dxtools Attribute Check Trigger is used to connect an check event on a Attribute in ENOVIA with a series of 3dxtools Activities to execute when the event fires.  Some examples of the operations you can perform using these triggers are:

•Use the Modify Document Attribute action when updating document approval comments.

Configuration and Settings

To setup a 3dxtools Attribute Check Trigger:

1. Add the Event Trigger to the Attribute in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Attribute you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the attribute you wish to use by querying like  below,

  MQL>list attribute ${Attribute_Name};

MQL will list the attribute name.

•On the Triggers tab for that attribute find the ENOVIA trigger you wish to run the 3dxtools Activities on.
•To find the triggers which are already existing on attribute write the below query,

  MQL>print attribute ${Attribute_Name} select trigger;

  This will print already existing triggers for the attribute.

   Example: Print attribute "Approval1 Comments" select trigger;

 

  The MQL result shows there are no trigger added on attribute,

•To add new trigger to the attribute
•Edit the trigger.
•Set the following values:

Name

Value

Notes

Action

emxTriggerManager

For details on emxTriggerManager please see the standard ENOVIA documentation

Input

Append to any existing value

AttributeAllDRToolsV6RunCheckTrigger

AttributeAllDRToolsV6RunCheckTrigger runs the dependant 3dxtools Activities immediately.

Trigger_Type

Name of the Trigger Type :Check

List of Events: Check,Action,Override

Event_Name

Name of the Event:Create

Modify,Create,Revise,etc..please see the standard ENOVIA documentation for more events

      to add new input to the event(Modify) which already exits for the trigger type(Check),

Syntax:

MQL>mod attribute ${Attribute_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Note:If  trigger already exists for ModifyCheck then append the new trigger and existing trigger with comma separator,

    Example:

    MQL>mod attribute "Approval1 Comments" add trigger modify Action emxTriggerManager input AttributeAllDRToolsV6RunCheckTrigger;

Confirm the trigger added  from the below query:    

   MQL>print attribute "Approval1 Comments" select trigger;

 

•Remove trigger:

  To remove trigger from the attribute mention the event and trigger type, it removes all the trigger which got added

  Before removing the trigger ,make a note of trigger which should not be removed for the event,so that it can be added later

   MQL>mod attribute "Approval1 Comments" remove trigger Modify Check;

 

2. Add the drV6tools Attribute Check Trigger Object ➣

•Add a suitably named 3dxtools Folder, or browse to an existing one.
•Select the Folder Content tab.
•Click on Actions and select Create DRLTools Type Check Trigger...

oThis will open the DRLTools Type Check Trigger create form.
•Set the following values:

Value

Type

DRLTools Type Check Trigger

{Operation Name} should be set to the trigger in ENOVIA you wish to run the activities from (for example ModifyAttribute, Checkin, etc)  

{Type Name} should be set to the ENOVIA object type the attribute is being set on.

{Attribute Name} should be set tot he ENOVIA attribute name the trigger is being run on.
