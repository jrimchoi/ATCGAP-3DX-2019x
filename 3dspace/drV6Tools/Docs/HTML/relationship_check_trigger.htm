Relationship Check Trigger


Contents

The 3dxtools Relationship Check Trigger is used to connect an check event on a Relationship in ENOVIA with a series of 3dxtools Activities to execute when the event fires. Some examples of the operations you can perform using these triggers are:

•Use the Remove Approval Route action when removing route in document lifecycle.

 

Configuration and Settings

To setup a 3dxtools Relationship Check Trigger:

1. Add the Event Trigger to the Relationship in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Relationship you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the relationship you wish to use by querying like  below,

  MQL>list relationship ${Relationship_Name};

 

MQL will list the relationship name.

•On the Triggers tab for that relationship find the ENOVIA trigger you wish to run the 3dxtools Activities on.

  To find the triggers which are already existing on relationship write the below query,

  MQL>print relationship ${Relationship_Name} select triggers;

   This will print already existing triggers for the mentioned relationship including both inherited triggers(inherited from parent relationship) and immediate triggers(added on this relationship)

   Example: Print relationship  "Object Route" select trigger;

 

The MQL result will list the triggers which are already added

    trigger = DeleteCheck:emxTriggerManager(RelationshipObjectRouteDeleteCheck)

    here,the trigger event is Delete and trigger type is Check (for more details on events and trigger type refer below table).

    for the DeleteCheck already a trigger named RelationshipObjectRouteDeleteChcek,RelationshipAllDRTollsV6RunCheckTrigger has added.

MQL>print relationship ${Relationship_Name} select immediatetrigger;

  This will print already existing triggers for the mentioned type which are directly added on the relationship.

•To add new trigger to the type
•Edit the trigger.
•Set the following values

Name

Value

Notes

Action

emxTriggerManager

For details on emxTriggerManager please see the standard ENOVIA documentation.

Input

Append to any existing value

RelationshipAllDRToolsV6RunCheckTrigger

RelationshipAllDRToolsV6RunCheckTrigger runs the dependant 3dxtools Activities immediately.

Trigger_Type

Name of the Trigger Type :Check

List of Events: Check,Action,Override

Event_Name

Name of the Event:Delete

Modify,Delete,Create,Revise,etc..please see the standard ENOVIA documentation for more events

to add new input to the event(Delete) which already exits for the trigger type(Check),

Syntax:

MQL>mod relationship ${Relationship_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Note:If immediate trigger already exists then append the new trigger and existing trigger with comma separator,

MQL>mod relationship "Object Route" add trigger Delete Check emxTriggerManager input RelationshipObjectRouteDeleteCheck,RelationshipAllDRToolsV6RunCheckTrigger,RelationshipAllDRToolsV6RunCheckTrigger_Defered;

Confirm the trigger added  from the below query:

     MQL>print relationship "Object Route" select trigger;

   

•Remove trigger:

To remove trigger from the type mention the event and trigger type, it removes all the trigger which got added

     Before removing the trigger ,make a note of trigger which should not be removed  for the event,so that it can be added later

     MQL>mod relationship "Object Route" remove trigger Delete Check;

   

    MQL>print relationship "Object Route" select trigger;

   

 it removed all the trigger which got added for createAction,So add the trigger which was not supposed to be removed,

    MQL>mod relationship "Object Route" add trigger Create Action emxTriggerManager input RelationshipObjectRouteDeleteCheck,RelationshipAllDRToolsV6RunCheckTrigger;

 

2. Add the drV6tools Relationship Check Trigger Object ➣

•Add a suitably named 3dxtools Folder, or browse to an existing one.
•Select the Folder Content tab.
•Click on Actions and select Create DRLTools Relationship Check Trigger...

oThis opens the DRLTools Relationship Action Trigger create form.
•Set the following values:
