Validate Route Content At Lead Engineer Approval


Contents

There should be a validation to validate documents which satisfy certain conditions before task approve or reject. Following are the basic validations configured using action activity:

•Check if approval route template to use for next node is define on document
•Check for attribute value i.e. Document Subtype, DR Use Template etc

 

1. Business Object and Execution  ➣

Name

Description

Route Setting

DRLTools Route Setting_UsersAtLeadEngineerApproval_1

Trigger Object

DRLTools Type Check Trigger_Route Setting Validation Placeholder_1. Details at Validation Objects

This is placeholder object which allow us to create actions to be executed to validate route content.

Action Activity

DRLTools Action Activity_ValidateRouteContentAtLeadEngineerApproval_1. Details at Validation Objects

Execution

For each selected document of DRL DOCUMENTS type system will run if condition for Document Subtype and DR User Template attributes. If Document Subtype is not empty and DR Use Template is set to Yes then it will run check condition for Approval Route. It must be filled in else operation will be blocked and redirected to validation report page.

Failure Message

If route content does not satisfy the defined criteria then system will block the operation and redirect control to validation report page
