If Condition Object


Contents

Each If Condition Activity can be linked to one of more If Condition Objects. Each of these objects contain a single condition to check. If all the conditions evaluates to true then any dependant 3dxtools Activities are carried out, if it evaluates to false then the system moves to the next item at this level.

The If Condition objects define a left and right hand side values, the sources of where to get them from and a condition to compare them with.

For example: Does the attribute “Owner” equal the current user name?

In this case the left hand source is “attribute” with a left hand value of “Owner”, the condition is “=” and the right hand source is “user admin object” with a right hand value of “name”

To create an If Condition object:

•Select the If Condition Activity Conditions tab on the If Condition Activity object
•Click Actions.
•Select Create DRLTools If Condition Object...

oThis will bring up the If Condition Object create form.
•Set the following values:

Left Hand Source

This defines the source value for the left hand side of the comparison and can be set to one of the following values. Depending on what is selected in this value drives the value that should be entered into the Left Hand Value attribute.

Left Hand Source value

Description

attribute

This gets a value from the current context object and can be any basic or attribute on the object. If the object does not have the attribute defined then this value will be defaulted to blank.

fixed value 

This uses the value in Left Hand Value as a fixed value.

mql selectable 

This allows you to use any valid MQL selectable from the current context object.

user business object 

This allows you to pull any basic or attribute value from the current users Person business object.

user business object mql selectable 

This allows you to use any valid MQL selectable from the current users Person business object.

user admin object 

This allows you to use any MQL selectable from the current users Person administrative object.

jpo

This allow you to use a JPO to retrieve the left hand value for the condition. It is useful for situation where the conditions you wish to check are more complex than the 3dxtools can handle.

 

file

This option is available for situations where the system is processing a file and allow you to compare the filename, fileextension and format

Left Hand Value

This value defines the left hand value to be used, based on and valid for the Left Hand Source value that has been selected. I.e if the Left Hand Source value equals attribute it should be either a basic or attribute name, if mql selectable it can be any valid mql selectable statement e.g. attribute[Net Part Weight].unitvalue[lb].

If Left Hand Source  is set to fixed value and the Condition is set to equals or not equals, then it is also possible to use wildcard characters in the Left Hand Value. The system accepts * for any number of any characters or ? for any single character.

if Left Hand Source  is set to file then the Left Hand Value. should be set to either fileName, format, fileExtension or fileNameWithoutExtension

if Left Hand Source  is set to jpo then the Left Hand Value. should be set to the {JPO Name}.{JPO Method Name} i.e. testJPO.testMethod

Below shows an example of the JPO code that can be used to calculate the value.

public class ${CLASSNAME}

{

   public String testMethod(Context context, String[] args) throws Exception {

        try{

            String returnValue = "";

            HashMap programMap  = (HashMap)JPO.unpackArgs(args);

            String objectId = (String)programMap.get("objectId");        

            //business logic to calculate returnValue          

         return returnValue;

        }catch(Exception ex){

            throw ex;

        }

    }

}

 

 

Condition

The condition type to use to compare the left and right hand values with. This value can be set to one of the following options.                      

Condition value

Description

= or Equals

Equal to

!= or Not Equals

Not equal to

< or Less Than

Less than (should only be used on numeric/date values)

<= or Less Than or Equal To

Less than or equal To (should only be used on numeric/date values)

> or More Than

More than (should only be used on numeric/date values)

> or More Than or Equal To

More than or equal To (should only be used on numeric/date values)

regex match 

Regular expression string match. If set to this then Right Hand Source should be set to expression and Right Hand Value should be set to a valid regular expression statement.

regex not match 

Regular expression string does not match. If set to this then Right Hand Source should be set to expression and Right Hand Value should be set to a valid regular expression statement.

contains

Test to see if the left hand value contains the right hand value (should only be used on string values).

does not contain 

Test to see if the left hand value does not contain the right hand value (should only be used on string values).


Right Hand Source

This defines the source value for the right hand side of the comparison and can be set to one of the following values. Depending on what is selected in this value drives the value that should be entered into the Right Hand Value attribute.

Right Hand Source Options

Description

attribute

This gets a value from the current context object and can be any basic or attribute on the object. If the object does not have the attribute defined then this value will be defaulted to blank.

expression

If the Condition is set to regex match or regex not match then this value must be used.

fixed value 

This uses the value in Right Hand Value as a fixed value.

mql selectable 

This allows you to use any valid MQL selectable from the current context object.

user business object 

This allows you to pull any basic or attribute value from the current users Person business object.

user business object mql selectable 

This allows you to use any valid MQL selectable from the current users Person business object.

user admin object 

This allows you to use any MQL selectable from the current users Person administrative object.

Right Hand Value

This value defines the right hand value to be used based on and valid for the Right Hand Source value that has been selected. I.e if the Right Hand Source value equals “attribute” it should be either a basic or attribute name, if “mql selectable” it can be any valid mql selectable statement e.g. “attribute[Net Part Weight].unitvalue[lb]”

If Right Hand Source  is set to fixed value and the Condition is set to either equals or not equals then it is also possible to use wildcard characters in the Right Hand Value. The system accepts * for any number of any character  or ? for any single character.
