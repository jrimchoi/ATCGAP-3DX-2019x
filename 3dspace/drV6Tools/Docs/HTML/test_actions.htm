How To Test Actions


Contents

The 3dxtools Test Actions provides the ability to run set of actions on any business object in the system. This is used to validate any structure before going through the procedure by defining actions configuration and environment variables

 

Test Actions Command

The 3dxtools Test Actions command will be available in the Actions menu on DRLTools Action Activity object.

This will bring up the test form below

 

Test Form

 

1. Field Details  ➣

Field

Description

Start Object

This is a mandatory field. This box should allow either a TNR or Object ID to allow the users to set the start object. OR

The search button should allow them to search and pick the start object

Note: Search type chooser will allow to search types defined in emxFramework.GenericSearch.Types key of emxSystem.properties file.

If TNR or ID entered manually and it is incorrect or business object does not exist in the system then it will through exception to the user

Environment Variables

This is optional field. You can define any required environment variables here.

Start Test

Clicking this button will run the test and then display the results in the Results Form below.

Close

Clicking this button will close the test form window

Results Form

1. Column Details  ➣

Field

Description

Level

Actions execution order

Action Name

Actions Name which is executed on the object

Status

Result for the actions

Action executed with Passed result.

 

  Action is of “if” category and executed with NotPassed result. Light green represents the non-mandatory condition\actions. It will not execute the child actions but continue executing other parallel actions.

 

Action executed with FailAndContinue result.

 

Action executed with FailAndStop result.

Action Type

Type of action which is executed

Log

Detailed log statement during the process. This will also display Fail Message at the end if action return failure

 

 

Below columns will provide the link to open the business object

Action Activity

Activity object used to get actions configuration and execute them

Current Object

Context object used in actions

Source Object

Source object which triggered the action

Parent Object

Parent object in any action if dependant activity is executed
