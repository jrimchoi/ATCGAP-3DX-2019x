Cascade Activity


Contents

The 3dxtools Cascade Activity allows you to provide simple classification systems at the click of a button. The Cascade consists of a multi-level pick list control that presents options to the user based on the currently selected or previous values.

The activity can be used to add or update ENOVIA objects, providing standard descriptions, filling in default values and providing custom unique mask ID numbers. The system can also be used to drive other ENOVIA functionality  by allowing it to build dynamic URLs based on the user selection, which is then forwarded to the ENOVIA web user interface to that URL.

 

Configuration and Settings

To setup a 3dxtools Cascade Activity:

1. Add the Maping for Struts2 in ENOVIA ➣

•The Cascade functionality is based on Struts2 framework.In order to use it in Enovia below configuration is required in web.xml .

<filter>

       <filter-name>struts2</filter-name>

      <filter-class>org.apache.struts2.dispatcher.ng.filter.StrutsPrepareAndExecuteFilter</filter-class>

   </filter>

   <filter-mapping>

       <filter-name>struts2</filter-name>

       <url-pattern>*.action</url-pattern>

   </filter-mapping>

 

 

2. Add the Command for using Cascade Activity in ENOVIA ➣

•In ENOVIA Business create a Command with suitable name and description for using Cascade action with a specific activity as described in Command Action Trigger
•On the Link tab in the Command set the following values:

Name

Value

Notes

Href

${ROOT_DIR}/3dxtools/drCascade.action?drtoolsKey={3dxtools Cascade activity name}&action=init&mode={type of action)

Where {3dxtools Cascade activity name} is the name of the 3dxtools Cascade Activity object and {type of action) is the action to be performed by Cascade functionality which is Add or range to be provided in the URL.

 

 

•Add the new Command to a Menu Object in Enovia as described in Command Action Trigger.

3. Add a drV6tools Cascade Activity object ➣

•Select the Dependent Activities tab on a top level 3dxtools Trigger action object.
•Click on Actions.
•Select Create DRLTools Cascade Activity...

oThis will open the DRLTools Cascade Activity create form.
•Set the following values:

Attribute Name

Value

Notes

Type

DRLTools Cascade Activity

This should already be set for you.

Name

{A suitable identifier}

Revision

{Any suitable value}

Relationship to Source Object

{Relationship name}

Applicable if Mode is Add. {Relationship name} is the relationship to use to connect the current context object with the newly created object.  If Relationship to Source Object is defined, Relationship to Source Object By Type is not used.

Relationship to Source Object By Type

Each line should consist of:

{Relationship name|Type}

Applicable if Mode is Add. {Relationship name|Type} is the relationship to use to connect the current context object with the newly created object, per type of source object.  It is only used if Relationship to Source Object is not defined.

Relationship is From Source Object

TRUE

or

FALSE

Applicable if Mode is Add. This value defines which of the objects at either end of the relationship should be set as the current context object.

If TRUE the relationship is from the source object.

If FALSE the relationship is not from the source object (and is therefore from the current context object).

Mode

Add or Update or Forward or range

Add means that the Cascade Activity will add an object.

Update means to update attributes of the business object.

range means to assign values to Webform attributes using cascade.

Forward means the Cascade Activity will forward the selections to the URL defined in Target URL.

Target URL

{URL}

Applicable if Mode is set to Forward. {URL} is the URL that the Cascade Activity will forward to.

Values To Append To Target URL

{URL parameters}

Applicable if Mode is set to Forward. {URL parameters} are the parameters to be appended to the URL defined in Target URL (and will usually be names of form controls).  Multiple parameters must be comma-separated.

Header

{Header text}

{Header text} is the header text displayed to the user at the top of the Cascade.

Sub Header

{Sub Header text}

{Sub Header text} is the sub header text displayed to the user underneath the Header Text.

Side Panel Text

{Side panel text}

{Side panel text} is optional text that can be displayed at the side of the Cascade.  Displaying it is controlled by Show Side Panel.

Show Side Panel

TRUE

or

FALSE

4. Add the Cascade Selection objects ➣

The Cascade Selection objects define the controls that are displayed in a Cascade, allowing the administrators to design the Cascade front end.

The example below shows a Cascade Selection Free Text object, which defines a Cascade Selection Free Text control.

To add Cascade Selection objects follow the steps below:

•Select the Cascade Selection tab on the DRLTools Cascade Activity.
•Click Actions
•Select one of the Create Cascade Selection Object... commands.

 

Each Cascade Selection object defines a different type of control, allowing administrators to design a versatile front end for the Cascade.

Cascade Selection Object

Description

Cascade Selection Date

Adds a calendar control.

Cascade Selection Free Text

Adds a text box control.

Cascade Selection Group

Adds a group of controls.

Cascade Selection List

Adds a list of selectable items.

Cascade Selection List Option

Represents a list item in a Selection list control.

Cascade Selection Search List

Allows to create a search query defined in the configurations and the results that match the query are returned as a list.

Cascade Selection Simple List

Represents a simple list of fixed values defined in the configuration and displayed as selectable items.
