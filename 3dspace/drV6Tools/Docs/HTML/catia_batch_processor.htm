CATIA Batch Processor


Contents

The 3dxtools CATIA Batch Processor is designed to run CATIA V6/3DExperiance Platform batch utility jobs and then handle the processing of any files resulting. As an example the system can be configured to on the promotion of a CATIA Drawing 3dxtools can create a Queue Object for this processor, which run can pick up the drawing, generate and execute a batch job to create a PDF and then save the resulting PDF back to ENOVIA as a derived output.

Prepare the Environment  ➣

On the machine that is to run the Job Server the correct version of CATIA should be installed and fixpacked to match the version of the ENOVIA MCS Installation.

Once installed and confirmed to be functional you should find and note down the full file path to the CATStart.exe application.

Next on the Job Server machine a system environment variable of PLMBATCH_NoPLMIDStream needs to be added with a value of 1. This will stop the CATIA Batch Management tool from serializing the batch job settings when saving to an XML file and allow the modification of the settings to meet the processor requirements.

 

Setup CATIA PDF Printer  ➣

If this processor is to be used to generate PDF's from CATIA drawings then a PDF printer needs to be added to the printers list. This is done using the Settings Management application

When the application is loaded select the Printers tab on the General settings. Click on the Add button next to the Printer List, select the 3DS Virtual Printer option and click OK.

This will bring up the Printer Properties page for the new printer to add. Set the Name to CATIA PDF Printer and the Driver to PDF (Portable Document Format). If required set the Maximum Paper Format. Once complete click the OK button to add the Printer to the list and then click the OK button to close the application.

 

Generate a template CATIA Batch XML File  ➣

For the processor to work it requires a template CATIA Batch XML file to use as the basis for the job. The system will then use this file, change the object and file path details to match the requirements of the object being processed and then run the job. The batch XML file is created using the Batch Management application and contains all the CATIA settings required to run the job, including the ENOVIA connection details, object search criteria and pattern and the specific details for the Batch job. When logging into the Batch Management application you MUST use the same user and the Job Server itself. This user must also be either a Project Leader in every collaborative space which is to contain objects to be processed or a System Administrator.

 

Generate a template CATIA Batch XML File for PDF Generation

Below shows an example of how to create a template XML file for the batch generation of PDF files from CATIA drawings.

Once logged in to the Batch Monitor application (using the same user name as the one used for the 3dxprocessor Job Server) double click on the PLMPrintBatch utility. Using the PLMPrintBatch utility configuration window select the following options.

•Select the CATIA PDF Printer (created in the previous section for preparing the environment above).
•Tick Print to file checkbox and define a temporary file name.
•Tick Print to one file checkbox.
•Add one business object to the list of objects to be processed (this can be any object as it will be replaced with a variable value at a later step)

•Click the Printers Properties... button.
•In the Printing Properties dialog select All sheets range value on the Multisheets tab.
•In the Printing Properties dialog select the Page Setup... button.
•In the Page Setup dialog tick the Use image format checkbox.

Once configured save the PLMPrintBatch job details to file by clicking on the Save button.

This will create an XML file with all the details for the job. This can then be altered to meet the 3dxtools CATIA Batch Processor requirements and then saved into the Settings field in the CATIA Batch Processor Settings object. For instance the example below shows the major revision, minor revision and name values in the XML being switched for the correct macro values. Along with these the other object and file references in the XML must be changed to meet the requirements of the processor.

 

Add Create Queue Object Activity to Trigger  ➣

To initiate the process of a specific business object the Create Object Activity should be added to a 3dxtools Trigger object assigned to a suitable trigger within ENOVIA. This trigger could be a check-in operation or a promotion event to a new state on the drawing. Below shows an example of setting the system up to create a queue object when the drawing is promoted to the Frozen state.

The first step is to add the standard 3dxtools Policy State Action Trigger to the IN_WORK Promote event on then policy for drawings (VPLM_SMB_Definition). This will cause the system to run the 3dxtools when the drawing is promoted.

The next step is to add a the Type Action Trigger object as defined in Policy State Action Trigger. When the trigger is fired by ENOVIA 3dxtools will search for a Type Action Trigger with a name matching the event and type that the trigger was fired from. In this case it is for the promote event on the drawing policy state IN_WORK.

 

To the Type Action Trigger object a Create Queue Object Activity should be added with the Processor name value set to the name of the CATIA Batch Processor object which you wish to run it.

Add the CATIA Batch Processor Settings  ➣

The CATIA Batch Processor Settings object contain the details of an individual CATIA Batch Utility job. These objects are added to the CATIA Batch Processor under the Batch Processor Settings tab. From a single CATIA Batch Processor it is possible to run multiple different CATIA Batch Utility jobs on a business object in a specific order using the order number attribute.

To add a CATIA Batch Processor Settings object:

•Select the Batch Processor Settings tab on the CATIA Batch Processor object.
•Click Actions.

Add the Derived Output Generator Settings  ➣

The Derived Output Generator Settings object define how the system should handle any files produced as a result of running the batch process defined in the CATIA Batch Processor Settings object. For instance if the batch process is setup to run the PLMBatchPrint process to produce a PDF file, then this object tells the system what to do with it, including the type, name and revision patterns to use, the policy and the make up of the intended output file name. It is also possible to perform more complex attribute mappings to the derived output object using the Copy Values system, this will allow you to map values from the source object to the created derived output object simply by associating Copy Values Target Attribute objects to the Target Attributes tab.

To add a Derived Output Generator Settings object:

•Select the Derived Output Settings tab on the CATIA Batch Processor object.
•Click Actions.
•Select Create DRLTools Processor Derived Output Generator Settings...

This will open the Processor Derived Output Settings create form.
