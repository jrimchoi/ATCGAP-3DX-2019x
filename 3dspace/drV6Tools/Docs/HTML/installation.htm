Installation


Contents

This is the core 3dxtools package installed onto the MCS and it includes all the Configuration UI, Core JSP and JPO methods required by 3dxtools and its dependant packages (for instance the 3dxprocessor Job Server). It also includes all the 3dxtools Activities.

 

3dxtools are delivered in standard install packages aligned with specific 3DEXPERIENCE ENOVIA releases and service packs. All customers use the same software, eliminating development configuration issues. The steps below define the installation steps for 3dxtools core package, which should be installed on the ENOVIA MCS. For details of how to install the 3dxprocessor Job Server click on Installation.

 

To install 3dxtools:

 

1. Run the Installer ➣

Instructions

Screenshots
•Download the appropriate zip file from https://workspace.designrule.co.uk/.
•Extract the zip file to a convenient location.
•Run 3dxtools.Installer <<version info>>.exe

•Click Next.

•Select the components you want to install.
•Click Next.

•Browse to the Web Application Directory for your ENOVIA installation (e.g. C:\enoviaV6R2013x\server\distrib\enovia).
•Browse to the Server Home directory for your ENOVIA installation (e.g. C:\enoviaV6R2013x\server).
•Enter the hostname (e.g. localhost).
•Enter the ENOVIA user to run the installation as (e.g. creator).
•Enter the password for the ENOVIA user.
•Click Next.

•Browse to Java Development Kit directory (e.g. C:\Program Files\Java\jdk1.7.0_11).
oThis MUST be a 64-bit version of Java.
•Click Install.

•Once Complete Click Close.

 

2. Perform post-installation tasks ➣

•Restart the Tomcat service.
•Compile all 3dxtools programs using the following MQL command:

compile program dr* force;

 

3. Verify the install ➣

•The install creates a log file. On Windows 7 or Windows Server 2008, this is located at in C:\Users\<username>\AppData\Local\Temp\<numbered folder>\3dxtools.Installer\install.log.
•Check the log file for any errors.
