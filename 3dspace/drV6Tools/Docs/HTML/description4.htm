Description


Contents

In this walkthrough, we will setup the mechanism to configure the standard command icons displayed in the ENOVIA Workspace Actions column. It gives the administrator the ability to turn on or off the displayed controls and allows for the insertion of a 3dxtools Command Action Trigger based (if required) on conditional values.

This package contains the complete configuration including reference files and test plans required to customize document Actions column. The configuration will add ‘Printed by: <First Name> <Last Name> - <timestamp>‘ in footer section of stamped PDF file. It will also add the watermark for the document state except Released documents.

This configuration allows user to download a stamped PDF instead of the standard download and viewer icons. But to only do this if the object is managing or linked to an object managing a PDF file.

Following are the 3dxtools functionalities covered in this walkthrough:

1.Create DRLTools Folder object

The 3dxtools Folder is the place holder object for 3dxtools triggers. It is used to combine and display the configured activities for a standard ENOVIA operation in an expanded view.

 In this example, we will create a placeholder for ENOVIA type action triggers.

2.Create Type Action Trigger object

The 3dxtools Type Action Trigger is used to connect an action event on a Type in ENOVIA. It is also used to create placeholder for 3dxtools activity objects.

In this example, we will add placeholder for PDF Stamper and If Condition activity objects.

3.Create If Condition Activity object

This feature is used to create placeholder for If Condition Objects to be used in document operations.

 In this example, we will create a placeholder for pdf format and processed queue condition objects.

 

4.Create If Condition Object

This feature provides customized environment for the user to perform any operation based on the defined set of conditions.

In this example, we will use the pdf format and processed queue conditions to perform the operation available in document Actions column.

5.Create PDF Stamper Activity object

This 3dxtools feature allows PDF documents to be stamped or watermarked. It is configured with XML. In this example, we will use First and Last name of logged in user to stamp Printed By information in footer.

 It will also watermark the document state if document is not in Released state.

 

6.Update OOTB Actions Column Schema

This is to add/update the schema to user 3dxtools JPO for actions column.

In this example, document Actions will be displayed based on the checked in file format.
