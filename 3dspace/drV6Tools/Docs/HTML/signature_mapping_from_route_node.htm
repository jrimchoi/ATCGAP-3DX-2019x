Signature Mapping From Route Node


Contents

For each node in the ENOVIA route that requires a signature mapping a individual Signature Mapping From Route Node object needs to be created and linked to a parent Signature Mapping From Route Activity. Signature Mapping From Route Node object provides all the information required to identify a particular node on a Route and then controls the mapping of the values using the Copy Values Target Attribute from the Route Node to the object being sent through the Route.

•Select the Signature Mapping From Route Nodes tab on the Signature Mapping From Route Activity.
•Click Actions.
•Select Create DRLTools Signature Mapping From Route Node...

•This will open the DRLTools Signature Mapping From Route Node create form.
•Set the following values

Route Base State

This attribute value defines the method to allow the Activity to find the correct Route to pull the values from. Each Route is linked to the object to be processed using an Object Route relationship. On this relationship is the attribute Route Base State which the Activity will use to match this value.

Route Node Title

As a Route can be made up of many nodes to ensure the Activity maps the value from the correct one the system uses this value to search all the node links to find the Route Node link with the matching Title value. Once found this is the object the system will pull the values from.

Route Node Approval Status

To ensure the values only get copied across if the Node has been completed/approved the system uses this value to compare with the Approval Status value on the found Route Node link object. If it matches the system will let the values be mapped. If it does not match the system will skip the mapping.

The Signature Mapping From Route Activity uses the settings defined in this Signature Mapping From Node object to identify a particular node on a Route and pass the current object that you wish to map to (in the example above this would be the Sketch object) the Route, the Person and Route Node relationship that connects them  and passes them all into the Copy Values system. Using the Copy Values Target Attribute linked to the Signature Mapping From Route Node object allows you to control which attributes on the source object you wish to update.

 

To each Copy Values Target Attribute linked to the Signature Mapping from Node object you create one or more Copy Values Element objects to define where to pull to the values from.

Copy Values Element Source value options

Source Object used

object

This pulls the value from object the activity is being run from.

to

The pulls the value from the Person object linked to the Route object using the Route Node relationship that matches the criteria defined in the Signature Mapping from Node object.

from

This pulls the value from the Route object.

relationship

This pulls the value from the Route Node relationship that connects the Route and Person objects that matches the criteria defined in the Signature Mapping from Node object.
