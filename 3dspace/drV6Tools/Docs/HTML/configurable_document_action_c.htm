Configurable Document Action Commands


Contents

3dxtools provides a simple mechanism to configure the standard command icons displayed in the ENOVIA Workspace Actions column. It gives the administrator the ability to turn on or off the displayed controls and also allows for the insertion of a 3dxtools Command Action Trigger based (if required) on conditional values. For example below shows a system configured to show 3dxtools command action trigger to download a stamped PDF instead of the standard download and viewer icons. But to only do this if the object is managing or linked to an object managing a PDF file.

Configuration and Settings

To setup the 3dxtools Configurable Document Actions Commands:

1. Find and Backup the OOTB ENOVIA Table Admin Objects ➣

To enable the 3dxtools Configurable Document Actions Commands it is required to modify two out of the box ENOVIA Table admin objects. Before doing this it is recommended to take a backup of the tables before modifying them.

The tables are called APPFileSummary (this it the standard table used for the File summary list shown when viewing a single common document), TMCContentSummary (this is the standard ENOVIA workspace summary table), APPQuickFileSummary (this is the standard table used for quick file access from the standard search window) and APPDashboardUserDocuments (this is the standard table for the Dashboard home page)

To take a backup simply find, select and clone the admin objects within Business.

3. Modify the OOTB ENOVIA Table Objects to Add Settings ➣

•On the APPFileSummary, TMCContentSummary, APPQuickFileSummary and APPDashboardUserDocuments table admin objects, select and edit the column for the Document Actions.
•Add the following settings to configure which commands get displayed.

Setting Name

Value

Notes

dr show check in

{true or false}

Display the check in action icon. If blank or not set the default of true is used.

dr show check out

{true or false}

Display the check out action icon. If blank or not set the default of true is used.

dr show download

{true or false}

Display the download action icon. If blank or not set the default of true is used.

dr show download if activity false

{true or false}

If the system is configured to use conditions to see if the 3dxtools activity icon is active or not then this switch is useful to turn on the download action icon if the 3dxtools activity is turned off.

dr show viewer

{true or false}

Display the viewer action icon. If blank or not set the default of true is used.

dr show viewer if activity false

{true or false}

If the system is configured to use conditions to see if the 3dxtools activity icon is active or not then this switch is useful to turn on the viewer action icon if the 3dxtools activity is turned off.

dr show subscribe

{true or false}

Display the subscribe action icon. If blank or not set the default of true is used.

dr show activity

{true or false}

Display the 3dxtools activity action icon. If set to true it requires the dr activity name and dr activity icon values to also be set. If blank or not set the default of false is used.

dr activity name

{a valid Command Action Trigger 

object name}

This defines the name if the 3dxtools Activity that should be run when the command icon is clicked

dr activity icon

{a valid path to a gif image file}

This defines the command icon image to use. It must be a valid path to an image inside the web app.

dr activity tooltip

{a suitable description}

This defines the tooltip that will be displayed if the users hovers the mouse over the command icon.

dr activity conditions

{a list of If Condition Activity

objects to check delimited by '|'}

This defines the list of If Condition Activity object names to check to see if the 3dxtools command activity should be displayed. It is possible to define multiple names of If Condition Activity objects, if any of them evaluate to TRUE the system will display the command icon. If all of them evaluate to FALSE the icon will not be displayed.

dr show activity for file extensions

{a list of file extensions 

delimited by ,}

This defines a list of file extension. If the current object manages a file with one of these extensions then the system will display the command icon. If it doesn't contain a file with one of these extensions then the command icon will not be displayed.

dr show activity check derived output

{true or false}

If the setting dr show activity for file extensions has file extension values then, if this value is set to true, the system will also check the attached derived output for files that match the source objects file names but end with file extensions from the dr show activity for file extensions list.

dr show activity check for any derived output

{true or false}

This setting enables a check for any derived output with a matching file extension if the setting dr show activity check derived output is set to true and the system fails to find a matching derived output by matching the file name.
