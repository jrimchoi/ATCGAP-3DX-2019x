Create Document


Contents

In this walkthrough, we will setup the 3dxtools configuration required to define document properties automatically using configurable functionalities provided in 3dxtools package. You can use a standard ENOVIA operation trigger to configure create document process.

This package contains the complete configuration including reference files and test plans required to customize create document process.

When users create a new document this configuration will run through the customer specific activities and update the document properties based on lookup parameters. These properties include the following scenarios:

•Apply a document template and route template
•Set default attributes
•Assign pre-defined prefix to the document name
•Apply document lifecycle to be used in review and approval

 

This configuration will provide the flexibility to control document behavior using least number of inputs while creating the document. This will help in implementation of requirement changes related to document lifecycle, customer can easily setup the new review and approval processes.

It will simplify the product configuration changes as lookup values are controlled in a simple layout file.

Overall implementation time and data creation will be reduced as most of the document properties will be automated in lookup activities.

 

Following are the 3dxtools functionalities covered in this walkthrough:

1.Create DRLTools Folder object

The 3dxtools Folder is the place holder object for 3dxtools triggers. It is used to combine and display the configured activities for a standard ENOVIA operation in an expanded view.

 In this example, we will create a placeholder for ENOVIA Create operation.

2.Create Type Action Trigger object

The 3dxtools Type Action Trigger is used to connect an action event on a Type in ENOVIA.

 In this example, we will connect Create event to the DRL DOCUMENTS type in ENOVIA.

3.Create Lookup Values Activity object

The 3dxtools Lookup Values Activity is a simple mechanism to define the object properties automatically using pre-defined configuration file (.csv).

The lookup value file contains the deciding parameters and dependent properties of an object. You can define the layout details to retrieve these parameters from the object and based on the lookup output specific properties will be applied.

 

In this example, we will use Business Unit, Type and Document Subtype as deciding parameters and Policy, Promote To State, Prefix, Sequence, UK Route Template, UK Doc Template, USA Route Template, USA Doc Template as dependent properties.

 

4.Create Change Policy Activity object

This 3dxtools Change policy activity feature provides the simple mechanism to configure the environment to change the policy of the object based on different/multiple conditions. This will help user to control or manage the object based on the policy.

An object policy is one which defines the behaviors of the object how it can be managed and controlled. Users must be confident that it is being set to the correct value. The policy assignment could be at the point the object is created or at some later stage when the object gets to a certain point in its lifecycle.

Different processes will have different lifecycles. Lifecycle is defined on Policy of the object. In design phase of any object, it will have once set of lifecycle. When the design is submitted, and approved in Engineering side, the lifecycle of the object may be different than the design phase. So, during the approval of the design document, based on the condition mentioned on the Change policy activity, policy of the object can be changed.

 

In this example, policy of the object will be changed based on the Document Sub Type attribute value in the DRL Document objects.

 

5.Create Copy Values Activity object

This 3dxtools Copy values activity feature provides the simple mechanism to configure the environment to change the attribute values of the object based on different/multiple conditions. This will help user to update the attributes on the object on creation or during execution of some process on the object.

Attributes are the values, which will define the characteristics of the object. Attribute value of the object can be changed/updated during creation of the DRL DOCUMENT object. Similarly, during any process, attribute values can be updated on the object based on the condition from user.

Attribute values of the objects can be different based on the process or zones. For example, if the object has attributes like currency, the format can be changed based on the region during the process.

 

In this example, we will copy Approval Route values from lookup activity reference file.

 

6.Create If Condition Activity object

This 3dxtools If Condition Activity feature provides customized environment for the user to perform any operation during the creation of the DRL Document object based on the defined set of conditions. Basically this is activity controls the dependent activity to perform its operation if the condition mentioned in this is passed.

 

In this example, based on the Sequence mentioned in the look up values, the sequence is set on the object if it is not equals to empty.

 

7.Create Sequence Number Activity object

This 3dxtools Sequence Number activity feature provides user to customize ENOVIA to update the sequence of the newly created/processed DRL Document object.  Basically, this feature is used to define/update the sequence of the object. For example, the sequence if the object can be 1 by default. If user wants to have it as A or any other sequence as default or on some action, it can be achieved using this feature of 3dxtools.

 

In this Example, the sequence of DRL DOCUMENT object is updated on creation based on the configuration defined in the Look up file.

 

8.Create Invoke JPO Activity object

This DRLTools Invoke JPO Activity feature provides simple mechanism to customize the Enovia environment to execute java program during the creation of DRL Document Object. Basically, this activity can be used to perform multiple operations like Promote, Change policy, set attributes, connect to multiple objects etc. To use this feature, administrator must insert the program to database as per Enovia coding standards.

 

In this example, DRLTools Invoke JPO Activity is used to promote the DRL DOCUMENTS object on Creation.

 

9.Create New From Template Activity object

The 3dxtools New From Template Activity defines the template file that should be checked in to the newly created DRL Document object. In this the conditions are checked to see if it should apply a template by checking if a file as already attached or not based. The file name for the new file by default is set to the current context objects name. For Example, this feature can be used to check-in in predefined document templates to the DRL Document based on the combination of Type, Sub type, policy etc.

 

In this Example, the document template is checked in to DRL DOCUMENTS objects during creation of the object based on the combination defined in Look up file.
