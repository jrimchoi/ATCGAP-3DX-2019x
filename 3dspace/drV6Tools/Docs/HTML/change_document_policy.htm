Change Document Policy


Contents

There should be a validation on policy change to check if the document has any active route. If incomplete approval process exists then policy change should not be allowed.

 

 

1. Business Object and Execution  ➣

Name

Description

Trigger Object

ChangePolicy_DRL DOCUMENTS. Details at Validation Objects

This is check trigger on DRL DOCUMENTS type.

Action Activity

DRLTools Action Activity_CheckRouteStatus_1. Details at Validation Objects

Execution

When user manually change the document policy the system will run forEach action on linked route and with if state check as In Process. If route is In Process then dependant check condition for Route Status will be executed. If any of the linked route is at In Process and Started then change policy operation will be blocked and an error message displayed to the user.

Failure Message

User friendly error message will be displayed on action failure
