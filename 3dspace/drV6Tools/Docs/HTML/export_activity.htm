Export Activity


Contents

The 3dxtools Export Activity lets you export and publish object metadata (in XML format) and files from ENOVIA into external systems (e.g. MRP, ERP and CRM systems) at any action trigger point. For example, when a Part is promoted to Production state the part metadata in XML format could be published to an Access database. The system provides a mechanism for defining what attributes, related objects and files (based on file extensions) should be exported. The system also provides several methods for defining the target location for the exported data including an external database, a dynamically named directory, or a file download to the web client machine. If set the file download and if the export contains multiple files the system will combine them into a single ZIP file for easy download to the users machine.

If Export Files is TRUE then this value is used to filter the list of files to export based on their file extension. Using this the system will only export the file if it has an extension matching one in the list. If this value is left blank then all files found will be exported.

Structured

or

Flat

This defines the layout of the exported XML.
 
Structured means objects will be organized in a hierarchically structured XML file, with sub objects nested inside the parent object.

Flat means the XML objects will be added as a flat (unstructured) list.

TRUE

or

Relationships To Include

This value is only used if Include Relationships is TRUE. This value allows the list of relationships to be included in the outputted XML metadata to be filtered to a specific list of relationship types defined in this list.  If left blank all relationships found will be included.

Types To Include

{List of Types,  one per line}

This allow for the types of objects being added to the outputted XML metadata to be filtered to a specific list of business object types defined in this list.  If left blank, all types will be included.

To enable this you must define this which defines the Type name and the Derived Output Relationship Name to enable the system to find the linked derived output objects.

Derived Output Relationship Name

{Relationship Name, 

either symbolic or actual}

This value is used in conjunction with the Derived Output Type Symbolic Name and enables the system to find the linked derived output objects.

Derived Output File Type

{List of File Extensions excluding the '.'

Delimited using a comma}

This defines the file types from the associated derived output objects that should be include in the export. The list should consist of the file extensions (excluding the proceeding dot) to include in the export. If left blank the system will get all the files

2. If required add any drV6Tools Export Type Setting objects ➣

To each Export Activity it is possible to link multiple 3dxtools Export Type Setting objects. These allow you to define more specific settings for the object types to be exported. Using these objects it is possible to define on a type by type basis the attributes to export. It is also possible to create more complex values using Copy Values Activity configuration by connecting Copy Values Target Attribute objects and  setting the Target Attribute value to the name of value you want to appear in the exported XML metadata.

3. Add the drV6Tools Export Target object ➣

The 3dxtools Export Target object defines targets for the metadata and files being exported by the Export Activity. Using these objects it is possible to configure the system to export to a fixed or variable directory location, a database that supports OLEDB connections, or as a HTTP file download if accessing though a command operation in the ENOVIA web environment.

 

To create an Export Target object:

•Select the Export Targets  tab on the Export Activity object.
•Click Actions.
•Select one of the Create Export Target commands.

The available 3dxtools Export Targets are:

• Export Target Database ➣

The 3dxtools Export Target Directory object exports the metadata selected as part of the parent Export Activity to any database that supports OLEDB. The XML metadata generated is passed into a defined large string field in a defined data table within the database. Other values can then be set using Copy Values Activity by attaching Copy Values Target Attribute and setting the Target Value attribute to the name of the database column you wish to set. This export target is typically used when you wish to use a database as the intermediary between the ENOVIA export and the import into a different 3rd party system. This allows you to use the intermediary database as a buffer between the two systems.

The system will by default use the 3dxtools XML export format to output the metadata of the selected exported objects. The system provides the ability to parse the XML file through an XML translation process by enabling the administrators to define a standard XML EXtensible Stylesheet Language Transformation (XSLT) setting. Details on XSLT can be found here: XSLT Tutorial.

 

To create an Export Target Database object:

• Export Target Directory➣

The 3dxtools Export Target Directory object exports the files and metadata selected as part of the parent Export Activity to a fixed or variable directory. The directory must be accessible from the MCS server and have both read and write access. This Export Target provides a very simple method of integrating with external systems such as MRP systems by have the 3rd party system monitor and pick up the files appearing in the export directory and importing them as required.

The system will by default use the 3dxtools XML export format to output the metadata of the selected exported objects. The system provides the ability to parse the XML file through an XML translation process by enabling the administrators to define a standard XML EXtensible Stylesheet Language Transformation (XSLT) setting. Details on XSLT can be found here: XSLT Tutorial.

 

To create an Export Target Directory object:

Overwrite Existing

TRUE

or

FALSE

This defines if the file should be overwritten if it already exists in the target export directory. If set to TRUE the system will overwrite the existing file with the newly exported one. If set to FALSE the system will check for the existing file and if found the system will append a counter number to the end of the file name (before the file extension) to make it unique.

Settings

{Additional Settings Values}

This setting defines the additional settings values. These are all optional and can be used on a as required basis.

ONE_OBJECT_PER_FILE={TRUE or FALSE} If set to TRUE the system will create a separate meta-data XML file for each object being exported.

SORT_ATTRIBUTES={TRUE or FALSE} If set to TRUE the system will sort the exported attributes by

putting the attribute names in alphabetical order.

 

SORT_RELATIONSHIPS={TRUE or FALSE} If set to TRUE the system will sort the child relationship objects

using the relationship name and the type, name and revision of the target object as the key to sort.

 

METADATA_FILE_NAME={File Name} If set this will be used for the file name for the meta-data xml file. 

This value will accept Macro Values.

 

• Export Target Download➣

The 3dxtools Export Target Download object provides the files and metadata selected as part of the parent Export Activity as an HTTP file download to the current users ENOVIA web session. This Export Target can only be used when the 3dxtools is run from a Command Action Trigger.  

The system will by default use the 3dxtools XML export format to output the metadata of the selected exported objects. The system provides the ability to parse the XML file through an XML translation process by enabling the administrators to define a standard XML EXtensible Stylesheet Language Transformation (XSLT) setting. Details on XSLT can be found here: XSLT Tutorial.

If the Export Activity results in multiple files then they will be added into a single ZIP file for download.
