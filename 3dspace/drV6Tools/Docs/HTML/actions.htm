Action

Description

Execution

If State Actions

These actions will be used to test the state of the current object. If the current object isn’t at the configured state then the action will not pass and further child actions will not be executed.

If state action will not block the current operation.

Check State Actions

These actions will be used to test the state of the current object. If the current object isn’t at the configured state then the action will not pass and further child actions will not be executed.

Check state action will block the current operation. You can define the failAction criteria to continue executing parallel actions or abort the operation.

If Type Actions

These actions will be used to test the type of the current object. If the current object isn’t of the configured type then the action will not pass and further child actions will not be executed.

If type action will not block the current operation.

Check Type Actions

These actions will be used to test the type of the current object. If the current object isn’t of the configured type then the action will not pass and further child actions will not be executed.

Check type action will not block the current operation. You can define the failAction criteria to continue executing parallel actions or abort the operation.

If Connection Actions

These actions will be used to test the number of connections to or from the current object that meet certain criteria. If the current object doesn’t have the defined connections then the action will not pass and further child actions will not be executed.

If connections action will not block the current operation.

Check Connection Actions

These actions will be used to test the number of connections to or from the current object that meet certain criteria. If the current object doesn’t have the defined connections then the action will not pass and further child actions will not be executed.

Check connections action will block the current operation. You can define the failAction criteria to continue executing parallel actions or abort the operation.

If Condition Actions

These actions will be used to test the current object using if conditions. The action object will contain a list of If Condition objects that if all passed or any one passed (based on the operator used) will then pass the action. If the current object doesn’t satisfy the defined conditions then action will not pass and further child actions will not be executed.

If condition action will not block the current operation.

Check Condition Actions

These actions will be used to test the current object using if conditions. The action object will contain a list of If Condition objects that if all passed or any one passed (based on the operator used) will then pass the action. If the current object doesn’t satisfy the defined conditions then action will not pass and further child actions will not be executed.

Check condition action will block the current operation. You can define the failAction criteria to continue executing parallel actions or abort the operation.

ForEach Actions

These actions will be used to test related objects of defined type that meet certain criteria. If the related object isn’t of the defined type then the further child actions will not be executed.

ForEach action will not block the current operation. But if any dependant check action returns failure then current operation will be blocked and child action’s validation message will be displayed to the user.

Get Actions

These Actions provides the ability to execute the actions on related object of context object. If the child action returns failure then the get action will not pass and further child actions will not be executed.

Get actions will not block the current operation. But if any dependant check action returns failure then current operation will be blocked and child action’s validation message will be displayed to the user

ActionActivity Actions

ActionActivity Actions provides the ability to to execute the independent action activity as a child action. If the action activity returns failure then the action will not pass and further child actions will not be executed.

ActionActivity actions will not block the current operation. But if any dependant check action returns failure then current operation will be blocked and child action’s validation message will be displayed to the user.

Variable Actions

Variable Actions provides the ability to maintain the list of variables used in action activity execution. The action object will contain a variable, operation and value to be used in operation. If any error occurs while executing actions then child actions will not be executed.

Variable action will not block the current operation.

If Variable Actions

If Variable Actions provides the ability to check the values of existing variable. The action object will contain a variable, operation ifVariable and value to be used in execution. If the defined value is not available in existing variable values then the action will not pass and further child actions will not be executed. If the operation is defined with ifVariable and action returns true then system will execute defined operation for the value.

If Variable action will not block the current operation.

Check Variable Actions

Check Variable Actions provides the ability to check the values of existing variable. The action object will contain a variable, operation ifVariable and value to be used in execution. If the defined value is not available in existing variable values then the action will not pass and further child actions will not be executed. If the operation is defined with ifVariable and action returns true then system will execute defined operation for the value.

Check Variable action will block the current operation. You can define the failAction criteria to continue executing parallel actions or abort the operation
