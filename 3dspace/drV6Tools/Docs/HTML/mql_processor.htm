MQL Processor


Contents

The 3dxprocessor Job Server MQL Processor is designed to run an MQL or TCL script as a batch operation, either locally in the Job Server machine or remotely communicating with the MCS. This processor is useful for running things like the full text search index command in a way that can be monitored from any administrator who can access ENOVIA. If you wish to run the script locally on the same machine as the Job Server itself then you must have ENOVIA Studio Modeling installed locally. To allow more configurable options the system also injects variables from the referenced object and any user required values (using the Copy Values Target Attribute) in the form of ENOVIA environment variables. These can then be called out and used in the TCL script.

 

Configuration and Settings

1. Add a drV6Tools Job Server MQL Processor object ➣

To add Job Server Activity Processor:

•Select the Processors tab on the Job Server Controller
•Click Actions.
•Select Create Job Server Processor...
•Select the Type Selector
•Select the Type Job Server MQL Processor

oThis will open the Job Server MQL Processor create form.

2. Add Any Require Copy Values Target Attributes ➣

To allow variables to be passed into the MQL program to be executed the system will create a number of ENOVIA environment variable. By default if the queue object running the process references a business object then the system will automatically add the following environment variables TYPE, NAME, REVISION, VAULT and OBJECTID, in addition to these any Copy Values Target Attribute connected to the processor will be also added as environment variables.
