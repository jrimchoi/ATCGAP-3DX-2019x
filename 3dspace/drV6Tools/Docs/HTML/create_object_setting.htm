Create Object Setting


Contents

To each Create Object Activity it is possible to link multiple 3dxtools Create Object  Setting objects. Each Create Object Setting defines the configuration required to create a single object. Including whether it should be created as new or cloned from the current object, what values to set and what relationships should be copied across form the current object to the new one. Once the object has been created it is also possible execute further operations on it by adding dependant 3dxtools Activities.

 

To create an Create Object  Setting object:

•Select the Create Object  Settings tab on the Create Object Activity object.
•Click Actions.
•Select Create DRLTools Create Object  Setting...

oThis will open the DRLTools Type Setting create form.
•Set the following values:

This defines if the system should clone the current object. If set to FALSE the system will create a brand new object instead.

If this value is set to TRUE then the Using Common Document Model value must be set to allow the cloning operation to be performed correctly.

Type Name

{New object type value}

This defines the type name to use for the new object.

Object Generator Name  

{Object Generator object name}

If you wish to use the next number in a standard ENOVIA sequence then this value must be set to the name value of an existing eService Object Generator object. It also requires the value Number Generator Name to also be set.

If this value is left blank then the system will use the name value defined in the setting Object Name.

Number Generator Name  

{Number Generator object name}

This defines the name value of an existing eService Number Generator object and must be defined if the Object Generator Name is set.

Object Name

{New object name value}

This defines the name to use for the new object. As well as being able to use the standard Macro Values to define the name this setting can also use two extras:

${counter} or ${counter:number of characters}

The counter macro can only be used at the end of the name and allows you to add a simple sequential counter number to the end of the name. The system will search the system for text marching up to the counter value, work out what the current maximum counter number is and then increment it by one.

${timestamp}

This is a simple way of creating unique ID using by allow the system to add the current system time in milli seconds as a timestamp to the name.

{New object revision value}

This defines the revision to use for the new object.

Change to Policy  

{New object Policy}

This defines the policy to use for the new object.

Export / Copy All Attributes

TRUE

or

FALSE

This defines if all the attribute values from the source object should be copied to the newly created one. This provide an alternative to using the Clone setting to copy the object as the clone operation may bring across extra unwanted relationships.

If you wish to copy only specific attribute values and/or you wish to calcualate and copy more complex values then you can do this using Copy Values system. This can be done by adding Copy Values Target Attribute objects to the Target Attributes tab on the Create Object Setting. The Target Value attribute on the Copy Values Target Attribute should be set to the attribute name on the newly created object that you wish to set.

Using Common Document Model

TRUE

or

FALSE

This defines if the source object uses the common document model and is required to be set to the correct value if the Clone value is set to TRUE.

Relationship To Source Object

{Relationship name}

This value allows you to define the relationship name and if set will create a connection of this type between the source object and the newly created one. If the relationship type changes depending on the source object type then the value Relationship To Source Object By Type can be used. If neither are set then no connection is created.

Relationship Is From Source Object

TRUE

or

FALSE

If the value Relationship To Source Object is set then this value allows you to control the direction of the link. If set to TRUE (ticked) then the relationship will originate from the source object to the new object. If set to FALSE (unticked) the the relationship will come from the new object to the source object.

Relationship To Source Object By Type  

Each line should consist of:

{Type=Relationship name}

This allows you to define a different relationship to create between the source object and newly created one based on the type of the source object. Where Type is the type name of the source object and Relationship name is the relationship to use.

Relationships To Include

Each line should consist of either:

{Relationship name}

or

{Relationship name}|to

or

{Relationship name}|from

This value defines the relationship type names of the relationships to be copied from the source object to the newly created object. If you specify just the relationship name then any relationship to the source object regardless of direction will be copied. If you want to only copy links of a specific direction then you can add "|to" or "|from" to the end of the relationship.

|from - should be used when the source object is at the from end of the relationship

|to - should be used when the source object is at the to end of the relationship

Export/Copy Expand Results

TRUE

or

FALSE

This define if along with the source object some/all of its related objects should also be copied at the same time. If set to TRUE the system will run an expand from the source object (using the Expand Statement Conditions as a filter if defined) and for each of the found objects check the dependent Copy Object Settings for the configuration to use to copy it.

{Valid MQL expand

The system controls the main MQL expand statement and only requires the expand statement conditions to be defined (the section highlighted in the MQL statement below).

Copy Related Objects

If Export/Copy Expand Results is set to TRUE the system will run an expand from the source object (using the Expand Statement Conditions as a filter if defined) and for each of the found objects check the dependent Create Object Settings for the configuration to use to copy it. To identify the correct Create Object Setting to use the system uses the following values on the relationship, matching them to the found objects relation type name and business object type name. The system loops through the list of available settings (using the Order Number  to control the order for the check), if the system finds a match it uses that setting to control creating a new object from the found one. If no setting is found then the object is skipped.

Dependent Create Object Setting objects can be added by selecting the Create Object Setting tab, clicking on Actions and selecting "Create DRLTools Create Object Setting...".

The system also allows values to be copied to the newly created object using the Copy Values system. This can be done by adding Copy Values Target Attribute objects to the Target Attributes tab on the Create Object Setting. The Target Value attribute on the Copy Values Target Attribute should be set to the attribute name on the newly created object that you wish to set.

Once the new object has been created, the values set and the relationships copied that system also allows for additional operations to performed on the new object by attaching any 3dxtools Activities to the Dependent Activities tab.
