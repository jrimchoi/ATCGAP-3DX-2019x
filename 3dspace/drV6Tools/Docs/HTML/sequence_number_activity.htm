Sequence Number Activity


Contents

The 3dxtools Sequence Number Activity extends the out-of-the-box ENOVIA auto-naming functionality, and allows you greater control over which number and object generator should be used for an object. Using 3dxtools you can use complex conditions to define which sequence should be used. For example, setting the sequence number based on a Document Sub Type attribute value.

 

Configuration and Settings

To setup a 3dxtools Sequence Number Activity follow the steps defined below.

1. Check/Add eService Number and Object Generator Objects ➣

•In ENOVIA Matrix, check to ensure that suitable eService Number Generator and eService Object Generator objects exist.
oIf they do not exist, add them (please see the standard ENOVIA documentation for details on how to do this).
•Note the names of the objects.

2. Add a drV6Tools Sequence Number Activity object ➣

•Select the Dependant Activities tab on either a top level 3dxtools Triggers action object or another 3dxtools Activities object.
•Click Actions.
•Select Create DRLTools Sequence Number Activity...

oThis will open the DRLTools Sequence Number Activity create form.
•Set the following values:

TRUE (ticked) means the system will use the background system super user to calculate the next value in the sequence. This is typically needed as usually standard users do not have the required permissions to update the sequence number.

FALSE (unticked) means the system will not use the background system super user to create the queue object.
