3dxprocessor Job Server Triggers


Contents

Job Server Trigger objects control when a 3dxprocessor Job Server will run and process a Queue Object. Its main purpose is to control how and when the trigger to retrieve or create the queue objects for specific Job Server Processors is fired, and then control the process of queuing the found/created queue objects up and passing them to running instances of the Job Server Processors.

 

To add a Job Server Trigger:

•Select the Triggers tab on the Job Server Controller object.
•Click Actions.
•Select one of the Create Job Server Trigger... commands.

 

Each of the creatable Job Server Trigger sub types provide different functionality to allow you to control how and when the trigger fires and are defined below:

Job Server Trigger Type

Description

Job Server Timer Trigger

Allows you to run a trigger based on a timer schedule (e.g. every hour)

Job Server Directory Watch Trigger

Allows you to run a trigger by watching a directory accessible from the Job Server for incoming files.
