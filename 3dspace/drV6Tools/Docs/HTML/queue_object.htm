Queue Object


Contents

Queue Objects are the mechanism that 3dxtools uses to queue jobs up for processing by the 3dxprocessor Job Server. It contains all the information the Job Server needs to carry out the task, including the object it should be running on, the requesting user, the file name and the specific name of the processor to do the job. The Queue Object also provides a mechanism to track the progress of the job through the processor including a status which highlights if a Queue Object is waiting to be processed, being processed or completed. It also include the process details which get updated live as the Queue Object is being processed, with the full report being added once complete.

Queue Objects are typically only created either by the Create Queue Object Activity or by one of the Job Server Triggers (if the trigger setting Queue Object Retrieval Type is set to Create New Queue Objects).

This value defines the status of the Queue Object. If it is set to Waiting then it is waiting to be processed by one the Job Servers. If it is set to Processing then its is currently being worked on by a processor.

If the task completes successfully then the status will be set to Processed. If a non-critical error occurs but the task still completes then the status will be set to Processed With Errors.

If there is a critical error and the task failed to complete then the value will be set to Processing Failed. With any Queue Object with this status the cause should be investigated and fixed. Once solved the Queue Object can have Status set back to Waiting to allow it to be reprocessed.

Abort Processing

TRUE

or

FALSE

This value provides a mechanism to abort a processor task once it has started. This can be useful if the processor becomes unresponsive or if the Queue Object has been stuck at Processing for too long a time (suggesting a problem with the processing job). Setting this value to TRUE will halt the processing and stop it from running in the future, until the value is set back to FALSE.

This value contains the debug output from the processor whilst it is processing the Queue Object. As the task is being processed for the Queue Object this value will be regularly updated (several times a minute) with the current status. Once the job is complete the full debug output will be stored in this field.
