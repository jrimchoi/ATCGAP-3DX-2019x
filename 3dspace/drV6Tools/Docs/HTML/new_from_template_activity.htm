New From Template Activity


Contents

It is recommended to use templates for all new file controlled objects in ENOVIA, particularly Office or CAD documents (e.g. Word, Excel, CATIA Drawings, etc.).  It will also be a requirement if setting up systems in ENOVIA that take advantage of ENOVIA integrations parameter mapping functionality (e.g. electronic signoff processes).

The 3dxtools New From Template Activity defines the template file that should be applied to the current context object. The system will check to see if it should apply a template by checking to see if a file as already attached or not. If a template is to be applied to system will get any business object linked with the DRLTools New From Template Activity Template Relationship and copy and apply any files it manages. The file name for the new file by default is set to the current context objects name, but it is possible to create more complex names using Copy Values Activity configuration by connecting a Copy Values Target Attribute object and giving it a Target Attribute value of FILENAME.

 

 

Configuration and Settings

To setup a 3dxtools New From Template Activity:

1. Add a drV6tools New From Template Activity object ➣

•Select the Dependant Activities tab on either a top level 3dxtools Triggers action object or another 3dxtools Activities object.
•Click Actions.
•Select Create DRLTools New From Template Activity...

oThis will open the DRLTools New From Template Activity create form.
•Set the following values:

This defines if the system should first check to see if any files are already being managed by the business object before adding the new file from the template.

Check For Files With Extension

{List of file extensions}

e.g.

docx,doc,xls,ppt,xlsx,pptx

If Check For Files is set to TRUE then this value can be used to check the file extension of any existing files on the current context object. If any match the file extensions defined here then the system will not add the template file. The file extensions should be entered without the preceding “.”. Multiple file extensions should be comma delimited with no following space.

2. Assign a Template Object to the New From Template Activity object ➣

Any object that can store and manage files in ENOVIA can be used to hold the template files used by the 3dxtools New From Template Activity. It is recommend to create a template class within the database and assign the permissions so only administrators can alter them.  To assign a template file for the activity follow the steps below.

•Select the New From Template Activity Templates tab on the 3dxtools New From Template Activity object.
•Click Actions.
•Select either Connect... to connect an existing template object, or Create {Document Type}... to create a new object of the correct document type.
