DRLTOOLS Initiate Route


Contents

 This object will be used to decide which route template will be used and provide different methods for defining it either by just specifying the route template name or by giving the route template relationship name to use to check for route templates linked to the currently selected objects.

 

 drlSettings – multiline string attribute

 

below shows an example of the XML setting that this attribute should contain:

 

 <settings>

 <routeTemplateRelationship>DRLTools Route Template Relationship</routeTemplateRelationship>

 <routeTemplateName>DMS*</routeTemplateName>

 <oneRoutePerObject>false</oneRoutePerObject>

 <createFormHeaderText>Start Approval Route</createFormHeaderText>

 <createFormTemplateLabelText>Approval Template</createFormTemplateLabelText>

 <noRouteFoundMessage>No Route Found.</noRouteFoundMessage>

 <noMatchingRouteFoundMessage>No Route that matches all objects found</noMatchingRouteFoundMessage>

 <autoStart>true</autoStart>

 </settings>

 

 routeTemplateRelationship: This setting is used to get matching route templates attached to the selected documents.

If the setting <routeTemplateRelationship> is defined, then this is the name of the relationship to use to find any linked Route Template objects attached to the currently selected business objects. If no route is found, then the message that should be displayed to the user can be found in the settings value <noRouteFoundMessage>. If multiple objects are selected then only the Route Templates that are associated with all selected objects should be displayed. If no Route Templates matching all objects are found then the error message defined in <noMatchingRouteFoundMessage> should be displayed.

 

 routeTemplateName: This setting is used to search the route template with the matching name. User can pass matching name (ex. *Approval*) to query the database.

 

If the setting <routeTemplateName> is defined then this is the name value to use to query the database for Route Templates with a matching name. If no route is found, then the message that should be displayed to the user can be found in the settings value <noRouteFoundMessage>.

 

 oneRoutePerObject: This setting will define whether one route must be created for each of the selected documents or one for all the selected documents.  If it is true then 1 route per document will be created. If false, 1 route will be created for all the selected documents.

This is an optional setting and default value is false.

 createFormHeaderText: This is the header used for create page of Start Approval Route process.

 

 createFormTemplateLabelText: This will contain the label required for Route Template combo box field on Start Approval Route page.

 

 noRouteFoundMessage: User alert for the single document selection based on results found using routeTemplateRelationship and routeTemplateName setting.

 

 noMatchingRouteFoundMessage: User alert for the multi document selection based on results found using routeTemplateRelationship and routeTemplateName setting.

 

 autoStart: This setting is used to define auto start of route at the end of Start Approval Route process. If it is true, Route will be started at the end of Start Approval Route process. If false, user must manually start the route.

This is an optional setting and default value is false.

 

Ex:

 

 <settings>

 <routeTemplateRelationship>DRLTools Route Template Relationship</routeTemplateRelationship>

 <routeTemplateName>DMS*</routeTemplateName>

 <oneRoutePerObject>false</oneRoutePerObject>

 <createFormHeaderText>Start Approval Route</createFormHeaderText>

 <createFormTemplateLabelText>Approval Template</createFormTemplateLabelText>

 <noRouteFoundMessage>No Route Found.</noRouteFoundMessage>

 <noMatchingRouteFoundMessage>No Route that matches all objects found</noMatchingRouteFoundMessage>

 <autoStart>true</autoStart>

 /settings>
