3dxprocessor Settings Manager


Contents

Each 3dxprocessor installation can handle multiple 3dxprocessor instances. Each of these instances create their own windows service that can connect to a 3DEXPERIENCE ENOVIA environment and run processing jobs. This allows for a single instance to connect to multiple 3DEXPERIENCE ENOVIA environment, which is particular useful in test and validation environments. The 3dxprocessor instances are created and managed using the 3dxprocessor Settings Manager application.

Each 3dxprocessor instance will create and mange it own windows service. You can view, start and stop the service either from inside the 3dxprocessor Settings Manager  or using the normal Services application.

The 3dxprocessor Settings Manager provides a series buttons

The functionality for these are described below.

Button

Description

Create

This button allows you to Create a Processor Instance by opening up the Create Form

Clone

This button will copy the settings of the currently selected processor instances in the grid and open the Create Form to allow you to edit them to match you new requirements.

Edit

If you wish to edit the settings of the currently selected processor instance in the grid then select it and then click this button. It will then open the Edit form for this item. You can also do the same operation by double clicking the processor instance in the the grid. Before the system opens the edit form it will stop the selected instances processor service.

Test

This button allows you to Test a Processor Instances 3DEXPERIENCE connection settings.

Delete

This will uninstall and delete the currently selected processor instance.

Start

This button will Start the currently selected processor instance windows service.

Stop

This button will Stop the currently selected processor instance windows service.

Refresh

This will refresh the details of the currently selected processor instance.

Switch View

Switch View allows you to switch into the protected desktop that the processor instances are running in. This can be useful as it allows you to see the actual processor instances running even though they are windows services.
