Relationship Action Trigger


Contents

The 3dxtools Relationship Action Trigger is used to connect an action event on a Relationship in ENOVIA with a series of 3dxtools Activities to execute when the event fires. Some examples of the operations you can perform using these triggers are:

•Use the Copy Values Activity to copy values between the objects at either end of the connection.
•Use the Send Email Activity to send a notification of the creation of the connection to particular users.

 

Configuration and Settings

To setup a 3dxtools Relationship Action Trigger:

1. Add the Event Trigger to the Relationship in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Relationship you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the relationship you wish to use by querying like  below,

  MQL>list relationship ${Relationship_Name};

MQL will list the relationship name.

•On the Triggers tab for that relationship find the ENOVIA trigger you wish to run the 3dxtools Activities on.

  To find the triggers which are already existing on relationship write the below query,

 MQL>print relationship ${Relationship_Name} select triggers;

           This will print already existing triggers for the mentioned relationship including both inherited triggers(inherited from parent relationship) and immediate triggers(added on this relationship)

           Example: Print relationship  "Active Version" select trigger;

The MQL result will list the triggers which are already added

    trigger = CreateAction:emxTriggerManager(RelationshipActiveVersionCreateAction)

    here,the trigger event is Create and trigger type is Action (for more details on events and trigger type refer below table).

    for the CreateAction already a trigger named RelationhsipActiveVersionCreateAction has added.

MQL>print relationship ${Relationship_Name} select immediatetrigger;

  This will print already existing triggers for the mentioned type which are directly added on the relationship.

•To add new trigger to the type
•Edit the trigger.
•Set the following values

Name

Value

Notes

Action

emxTriggerManager

For details on emxTriggerManager please see the standard ENOVIA documentation.

Input

Append to any existing value

RelationshipAllDRToolsV6RunActionTrigger

or

RelationshipAllDRToolsV6RunActionTrigger_Deferred

RelationshipAllDRToolsV6RunActionTrigger runs the dependant 3dxtools Activities immediately.

RelationshipAllDRToolsV6RunActionTrigger_Deferred waits until the current transaction scope has completed before running the dependant 3dxtools Activities.

Trigger_Type

Name of the Trigger Type :Action

List of Events: Check,Action,Override

Check: It performs an action before an event occurs.

Action: It performs an action by replacing an event.

Override:It performs an action after an event has occurred.

Event_Name

Name of the Event:Revise

Modify,Create,Revise,checkin.etc..please see the standard ENOVIA documentation for more events

to add new input to the event(Create) which already exits for the trigger type(Action),

Syntax:

MQL>mod relationship ${Relationship_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Note:If immediate trigger already exists then append the new trigger and existing trigger with comma separator,

MQL>mod relationship "Active Version" add trigger Create Action emxTriggerManager input RelationshipActiveVersionCreateAction,RelationshipAllDRToolsV6RunActionTrigger;

Confirm the trigger added  from the below query:

     MQL>print relationship "Active Version" select trigger;

   

• Remove trigger:

To remove trigger from the type mention the event and trigger type, it removes all the trigger which got added

     Before removing the trigger ,make a note of trigger which should not be removed  for the event,so that it can be added later

     MQL>mod relationship "Active Version" remove trigger Create Action;

   

    MQL>print relationship "Active Version" select trigger;

   

    it removed all the trigger which got added for createAction,So add the trigger which was not supposed to be removed,

    MQL>mod relationship "Active Version" add trigger Create Action emxTriggerManager input RelationshipActiveVersionCreateAction;

•Add a suitably named 3dxtools Folder, or browse to an existing one.
•Select the Folder Content tab.
•Click on Actions and select Create DRLTools Relationship Actions Trigger...

oThis opens the DRLTools Relationship Action Trigger create form.
