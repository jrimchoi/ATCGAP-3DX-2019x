Derived Output Generator Processor


Contents

The 3dxtools Derived Output Generator Processor is designed to generate derived output files for objects checked into the ENOVIA store. As part of the process the system will open the file in its native application and create outputs such as CGR, IGES and STEP.

 

The currently support native file formats are:

o CATIA V5 CATParts and CATProducts
o SolidWorks Parts and Assemblies
o Solid Edge Parts and Assemblies

 

Configuration and Settings

1. Add a drV6Tools Job Server Derived Output Generator object ➣

The main configuration for the Derived Output Generator is controlled using a Job Server Derived Output Generator object. Connected to this are Derived Output Generator Settings objects that hold the specific settings for each of the file types that can be processed. To add Job Server Derived Output Generator object:

•Select the Processors tab on the Job Server Controller
•Click Actions.
•Select Create Processor System Processor...
•Select the Type Selector
•Select the Type Processor Derived Output Generator

oThis will open the Processor Derived Output Generator create form.
•Set the following values:

2. Add the required Processor Derived Output Generator object values ➣

Each of the file types (CATIA, SolidWorks, etc) have their own configuration requirements to allow them to be processed. To accommodate this edit the newly created Processor Derived Output Generator object and enter the details required.

•Select the Derived Output Generator Settings.
•Click Edit.
•Set the following values:

CATIAv5: drV6Tools.NET.DerivedOutputGenerator.CATIAGenerator.drCATIAGenerator

Solid Edge: drV6Tools.NET.DerivedOutputGenerator.SolidEdgeGenerator.drSolidEdgeGenerator

Temp Path

{A temporary directory path}

This defines the temporary directory on the machine which the processor is installed on that will be used to hold all the temporary processing files. It must exist and the windows user running the process must have full access.

Use Command Line Execution

{TRUE or FALSE}

This defines if the system should use the default ActiveX activation for the solidworks application (set this value to FALSE) or use a specific command executable and arguments if required (set this value to TRUE). If set to TRUE you must define Command Line Executable and Command Line Arguments.

Command Line Executable

{Full path to CAD Application EXE}

If Use Command Line Execution  is set to TRUE then this value must be the full path to the application executable.

e.g.

CATIAv5: C:\Program Files\Catia\V5R27\win_b64\code\bin\CATSTART.exe

Solid Edge: C:\Program Files\Solid Edge ST9\Program\SolidEdgeTranslationServices.exe

SolidWorks: N/A

Command Line Arguments

{Any required command arguments}

If Use Command Line Execution  is set to TRUE then this optional value defines any command line arguments you may wish to pass when executing the application via the command line.

e.g. -run "CNEXT.exe" -env CATIA.V5-6R2017.B27 -direnv "C:\ProgramData\DassaultSystemes\CATEnv" -nowindow
