cell Tag


Contents

The cell tag holds the value for the cell, a number of tags can define how the information is going to be retrieved. The tag attributes column and row define which cell will be written to.  If cell is used inside a rowRepeater tag then row does not need to be defined. The cell tag has a series of possible sub tags: mqlCommand, copyValue, mql-Selectable, and value. These tags are what return a value which is then written to the report.

 

Example cell XML

This example would be used inside a list of fixed cells as the row is required.

 

<cell column="B" row="2">    
 <mqlCommand>version</mqlCommand>  
</cell>
<cell column="E" row="1">   
 <copyValue>BuildDate</copyValue>
</cell>

 

 

This example would be used inside a rowRepeater as there is no row defined.

<cell column="I">  
 <mql-selectable>relationship[Part to Plant Number].from.relationship[TLPN].from.attribute[Item Number Rating]</mql-selectable>   <!-- drXLReportMQLSelectableElement extends drXLReportElement -->
</cell>
<cell column="J"> 
 <value>name</value>  
</cell>
