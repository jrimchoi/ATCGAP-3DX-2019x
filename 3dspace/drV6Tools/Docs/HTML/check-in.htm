Check-in


Contents

 In this walkthrough, we will setup the 3dxtools configuration required to check-in the files to DRL DOCUMENTS objects which will intern create the que object for 3dxprocessor Job Server and name the checked in file based on the name, title and revision of the DRL DOCUMENTS object. Using this configuration, the checked in files can be easily renamed based on the DRL DOCUMENTS property and which will ease user to identify the document specific to the document. Also this will help user to maintain the particular pattern of naming of the document files. The que object creation will help the 3dxprocessor Job server to execute particular operation as per schedule. The queue object will be the reference to execute the operation to specific document object.

This package contains the complete configuration including reference files and test plans required to customize create document process.

This configuration will provide the flexibility to control document behavior using least number of inputs while check-in of files to the document. This feature of 3dxtools will reduce the input time during the data creation by large extent. This will help in implementation of naming conventions to be followed during upload of files to document and also helps in maintaining the scheduled tasks for each of check-in/upload activity.

Following are the 3dxtools functionalities covered in this walkthrough:

1.Create DRLTools Folder object

The 3dxtools Folder is the place holder object for 3dxtools triggers. It is used to combine and display the configured activities for a standard ENOVIA operation in an expanded view.

 In this example, we will create a placeholder for ENOVIA  check-in operation.

2.Create Type Action Trigger object

The 3dxtools Type Action Trigger is used to connect an action event on a Type in ENOVIA.

In this example, we will connect check-in event to the DRL DOCUMENTS within the Workspace folder.

3.DRLTools Create Queue Object Activity

The 3dxtools Queue Object Activity will create queue objects which is used by the 3dxprocessor Job Server to execute scheduled tasks on particular object. This Queue object will contain the information like object information, file name and name of the processor to do the job.

4.DRLTools Sync File Name With ID Activity

This 3dxtools Sync File Name With ID Activity used to enable the environment to change the names of incoming files to DRL DOCUMENTS object. This will help the user to maintaining the naming convention of files uploaded to DRL DOCUMENTS. This feature will help in identifying the files related to specific documents in Enovia environment.
