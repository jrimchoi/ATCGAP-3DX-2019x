Check Trigger Actions


Contents

A series of actions can be configured with check trigger in ENOVIA. These actions will be used to validate object during trigger event.

The section below shows the validation system triggers and actions configured in test platform for documents.

 

 

1. Configure Actions  ➣

Action

Description

Change Document Name

Validation to check if the document name is defined using lookup values. If lookup sequence number is configured then manually entered name not should be updated

Change Document Policy

Validation on policy change to check if the document has any active route. If incomplete approval process exists then policy change should not be allowed

Modify Document Attribute

Validation on attribute update based on related approval route tasks. If document approval comments are mapped with user comments during the approval process then manual update should be blocked if any incomplete mapping route task is available

Revise Document

Validation on document revision which allows us to check the status of related objects before revising the document

Remove Approval Route

Validation to check if the approval route is created to a specific state block or throw-out the lifecycle based on certain attribute condition on document

Promote Document

DMS Approval Tasks Check

Validation to block manual promotion of document if there are any route tasks configured for approval comments mapping and includes after approve action to promote the route content

Promote Document

Customer and Address Attribute Check

Validation to check document attributes before promoting.

For example, approval comments, customer details attribute should to be filled in during the customer approval process.

Promote Document

Approval Route Check

Validation to check approval route before promoting.

For example, If Document Subtype and Approval Route attributes are empty then there should to an approval route connected to the document
