Check Condition Actions


Contents

The 3dxtools Check Condition Actions provides the ability to test the current object using if conditions. The action object will contain a list of If Condition objects that if all passed or any one passed (based on the operator used) will then pass the action. If the current object doesn’t satisfy the defined conditions then action will not pass and further child actions will not be executed.

Check condition action will block the current operation. You can define the failAction criteria to continue executing parallel actions or abort the operation.

 

Configuration and Settings

To setup a 3dxtools Check Condition Actions:

1. Setting Configuration ➣

•Add high level business logic in below format to any Action Activity object Settings attribute.

<checkCondition>

 <name>Check Title And Description Are Not Blank</name>

<failAction>FailAndContinue</failAction>

 <failMessage>${Type} ${name} ${revision} title and description must be filled in</failMessage>

 <conditions>

 <condition leftHandSource="value" leftHandValue="title"

 condition="!=" rightHandSource="fixed value"

 rightHandValue=""/>

 <condition leftHandSource="value" leftHandValue="description"

 condition="!=" rightHandSource="fixed value"

 rightHandValue=""/>

 </conditions>

 <actions/>

</checkCondition>

 

<checkCondition>

 <name>Check Formal Release OR Customer Release Policy</name>

 <failAction>FailAndContinue</failAction>

  <failMessage>${Type} ${name} ${revision} must have Formal Release or Customer Release policy</failMessage>

 <conditions operator=”or”>

 <condition leftHandSource="value" leftHandValue="policy"

 condition="=" rightHandSource="fixed value"

 rightHandValue="Formal Release"/>

 <condition leftHandSource="value" leftHandValue="policy"

 condition="=" rightHandSource="fixed value"

 rightHandValue="Customer Release"/>

 </conditions>

 <actions/>

</checkCondition>

2. Setting Details  ➣

Setting

Description

name

Any meaningful value. It is recommended to start with “Check”.

failAction

Define the execution criteria if object doesn’t pass the configured check. It can be Passed, NotPassed, FailAndStop and FailAndContinue.

FailAndStop will block the operation and stop executing parallel actions.

FailAndContinue will block the operation and allow executing parallel actions.

Only FailAndStop and FailAndContinue actions will display error message to the user.

failMessage

User friendly error message to be displayed on action failure. You can use macros for basics and attribute values. Macros will be evaluated on context object id

conditions

List of condition be tested on current object.

operator

If “or” then condition will be executed using OR operator. If any one is true then action will be passed. It is optional parameter and default is “and” operator.

condition

Add any If Condition Object you wish to check for this event

actions

child actions configuration

 

•For child actions, add any Action Activity you wish to run from this event.
