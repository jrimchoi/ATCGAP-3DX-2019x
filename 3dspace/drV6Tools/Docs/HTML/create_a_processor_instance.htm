Create a Processor Instance


Contents

Clicking on the Create button on the 3dxprocessor Settings Manager brings up the Create form. Using this you can define all the details required to create a 3dxprocessor instance.

3dxprocessor Instance Settings

Processor Instance Name

This is the local name for the 3dxprocessor instance.

Processor Controller Object Name

This should match the 3dxprocessor Job Server Controller object in the 3DEXPERIENCE platform environment that is to hold and manage this 3dxprocessor instance.

Description

This is a description for the 3dxprocessor instance.

 

Main Settings Tab

ENOVIA Settings

Host

This is the URL to the 3DEXPERIENCE platform that this 3dxprocessor instance is going to connect to. It is recommended to check this URL works from the 3dxprocessor machine by testing it in a web browser using the credentials defined below.

User Name

The 3DEXPERIENCE platform user name that should be used to run 3dxprocessor instance. Please ensure this user has all the permissions and licenses required to run its processing jobs. This may require making it a system and business admin user or adding it as a leader in all collaborative spaces. If the 3dxprocessor instance is running on an FCS site then the user must have this site assigned as its location.

Password

The 3DEXPERIENCE platform users password.

Role

The 3DEXPERIENCE platform users role.

Vault

The default The 3DEXPERIENCE platform vault. Typically this is eService Production.

Using LDAP

If the The 3DEXPERIENCE platform is using LDAP for authentication then tick this box.

Using Passport

If the The 3DEXPERIENCE platform is using 3DPassport for authentication and you are connecting to the 3dspace host (and not internal) then tick this box.

Version

This defines the version of the 3DEXPERIENCE platform that you are connecting to.



Windows Settings

Domain

The windows domain for the windows login the 3dxprocessor instance will run under.

User Name

The windows user name 3dxprocessor instance will run under. This user must be a local administrator and have the permissions to Log on as a service.

Password

 

The windows users password.

Java Settings

Java JDK Path

This is the path to the top level folder of the Java JDK installed on this machine. It must be the 64bit version and it is also recommended that the JDK version matches the one used by the 3DEXPERIENCE platform.

 

Additional Settings Tab

Settings

Java Settings

This defines the settings that are passed to the java command line to execute the 3dxprocessor instance. Usually you will not need to change these from the default value. However if this instance is running memory intensive jobs (i.e. large report generation) you may need to increase the memory allocation here.

Startup Type

This defines the statup type for the 3dxprocessor instances windows service. The default is Automatic which means the service will startup automatically when the the machine starts up.

Notification Base URL

If the jobs being processed by the instance send notification messages (i.e. job complete notifications etc) then this value must be set to define the public access notification url.
