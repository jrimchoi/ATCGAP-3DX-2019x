Invoke JPO Activity


Contents

The 3dxtools Invoke JPO Activity can be used in situations where out-of-the-box 3dxtools cannot meet the customization requirement, and you need to add in your own code. It allows you to define the JPO program and method name and input a number of arguments. This allows you to plug in your own code anywhere it is required in the 3dxtools activity tree.

 

Configuration and Settings

To setup a 3dxtools Invoke JPO Activity:

1. Add/Find the JPO within ENOVIA ➣

Find (or if required create) the JPO with ENOVIA that you wish to invoke using this Activity, and note down the program object name and method name. For details on how to find or create the JPO please see the standard ENOVIA documentation. Ensure the JPO compiles properly before trying to invoke from a 3dxtools Activity.

Below is a template for the JPO method. Change the function name for something more suitable to the requirements:

    public int functionName(Context context, String[] args) throws Exception {

 
        //The ID of the object the 3dxtools Invoke JPO Activity is always the first thing in the args array.
        String objectID = args[0];

 
        //The optional values below are passed in from the 3dxtools Invoke JPO
        //Activity object settings.
        String arg1 = args[1];
        String arg2 = args[2];
        String arg3 = args[3];
 

        //Code Here

 
        return 0;
    }

2. Add a drV6tools Invoke JPO Activity object ➣

•Select the Dependant Activities tab on either a top level 3dxtools Triggers action object or another 3dxtools Activities object.
•Click Actions.
•Select Create DRLTools Invoke JPO Activity...

oThis will open the DRLTools Invoke JPO Activity create form.
