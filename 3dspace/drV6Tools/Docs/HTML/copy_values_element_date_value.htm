Copy Values Element Date Value


Contents

The DRLTools Copy Values Element Date Value allows you to calculate a date value using either a defined variable like the day of the current week or by pulling it as a a basic or attribute value from the object defined in the Source value.

To add Copy Value Element Date Value follow the steps below.

•Select the Copy Value Elements tab on the Copy Values Target Attribute.
•Click Actions.
•Select Create Copy Value Element Date Value...

oThis will open the DRLTools Copy Value Element Date Value create form.
•Set the following values:

Source

This defines the source value. Depending on what is selected in this value drives the value that should be entered into the Value attribute.

Source Options

Description

object

Retrieve the value from the current context object and the Value attribute value can be set any basic or attribute on the object.

user 

Retrieve the value from the current user object and the Value attribute value can be set any basic or attribute on the user object.

relationship 

Retrieve the value from the current context relationship and the Value attribute value can be set any basic or attribute on the relationship.

parent

Retrieve the value from the current context parent object and the Value attribute value can be set any basic or attribute on the parent object.

from 

Retrieve the value from the object on the from end of the current relationship and the Value attribute value can be set any basic or attribute on the object.

to

Retrieve the value from the object on the to end of the current relationship and the Value attribute value can be set any basic or attribute on the object.

Value

This value defines the basic or attribute name to be used to retrieve the value from the object defined in the drlSource. It must be a valid basic or attribute name and the value retrieved must be a valid date.

Value Options

Description

NOW

Uses the current server date and time

CLIENTNOW

Uses the current clients date time. NOTE: This function will only work if executed from a 3dxtools Command Action Trigger. If executed from any of the operation triggers like Type Action Trigger, Relationship Action Trigger, Attribute Action Trigger or Policy State Action Trigger then the current server date and time will be used.

{MONDAY, TUESDAY, 

WEDNESDAY, THURSDAY, 

FRIDAY, SATURDAY or SUNDAY}  

Uses the defined day of the current week. So by setting this value to MONDAY will set the system to use the date of the Monday of the current week.

DAYOFMONTH,{Number} 

Retrieves a specific day from the current month where {Number} is the number of the day from the month. i.e. DAYOFMONTH,1 will set the system to use the 1st of the current month.

DAYOFYEAR,{Number} 

Retrieves a specific day from the current year where {Number} is the number of the day from the year. i.e. DAYOFYEAR,1 will set the system to use the 1st of the current year.

Date Format

This value defines the formatting string to use to convert the date value into the presentation string format required. The system uses the formatting options in the standard Java class SimpleDateFormat for instance dd/MM/yyyy HH:mm:ss.

Setting this value to ENOVIA will set the format based on the current ENOVIA database format requirements.

Calculate Offset

This should be either TRUE or FALSE and defines if an offset value should be added to the date before formatting it. This will allow you to set the presented value to a predefined future or past date. If set to TRUE the system will add the values defined in Add Minutes, Add Hours, Add Days, Add Months and Add Years values.

It is possible to subtract time periods as well by setting the add values to negative numbers.

Add Minutes

This value must be an integer number and defines the amount of minutes to add to the date. Only used if Calculate Offset is set to TRUE. Setting this to a negative number will subtract that number of minutes.

Add Hours

This value must be an integer number and defines the amount of hours to add to the date. Only used if Calculate Offset is set to TRUE. Setting this to a negative number will subtract that number of hours.

Add Days

This value must be an integer number and defines the amount of days to add to the date. Only used if Calculate Offset is set to TRUE. Setting this to a negative number will subtract that number of days.

Add Months

This value must be an integer number and defines the amount of months to add to the date. Only used if Calculate Offset is set to TRUE. Setting this to a negative number will subtract that number of months.

Add Years

This value must be an integer number and defines the amount of years to add to the date. Only used if Calculate Offset is set to TRUE. Setting this to a negative number will subtract that number of years.
