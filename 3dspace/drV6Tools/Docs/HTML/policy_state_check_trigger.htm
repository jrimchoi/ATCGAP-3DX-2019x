Policy State Check Trigger


Contents

The 3dxtools Policy State Check Trigger is used to connect an check event on a Policy State in ENOVIA with a series of 3dxtools Activities to execute when the event fires. Some examples of the operations you can perform using these triggers are:

•Use the Promote Document Approval Route Check action when promoting document to Review or Approved state.
•Use the Promote Document Approval Tasks Check action when promoting document to WIP state.
•Use the Promote Document Customer And Address Attribute Check action when promoting document at Customer Review state.

 

Configuration and Settings

To setup a 3dxtools Policy State Check Trigger follow the steps defined below.

1. Add the Event Trigger to the Policy State in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Policy you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the policy you wish to use by querying like  below,

  MQL>list policy ${Policy_Name};

 

    MQL will list the policy name.

•On the Policy find the state you wish to run the trigger from.

 MQL>print policy ${Policy_Name} select state ${State_Name.trigger};

 Example: MQL>print policy "Formal Release" select state[Review].trigger;

The MQL result will list the triggers which are already added

    trigger = PromoteCheck:emxTriggerManager(TypeAllDRToolsV6RunCheckTrigger)

    here,the trigger event is Promote and trigger type is Check (for more details on events and trigger type refer below table).

    for the PromoteCheck already a trigger named TypeAllDRToolsV6RunCheckTrigger has added.

•To add new trigger to the Review state,
•Edit the trigger.
•Set the following values:

Modify,Create,Revise,etc..please see the standard ENOVIA documentation for more events

          to add new input to the event(Promote) which already exits for the trigger type(Check),

Syntax:

MQL>mod policy ${Policy_Name} state ${State_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Note:If  trigger already exists for ModifyCheck then append the new trigger and existing trigger with comma separator,

     Example:mod policy "Formal Release" state Review add trigger Promote Check emxTriggerManager input TypeAllDRToolsV6RunCheckTrigger,PolicyAllStateAllDRToolsV6RunCheckTrigger;

   

    Confirm the trigger added  from the below query:    

    MQL>print policy "Formal Release" select state[Review].trigger;

   

•Remove trigger:

  To remove trigger from the attribute mention the event and trigger type, it removes all the trigger which got added

  Before removing the trigger ,make a note of trigger which should not be removed for the event,so that it can be added later

 MQL>mod policy "Formal Release" state Review remove trigger Promote Check;

 

 MQL>print policy "Formal Release" select state[Review].trigger;

 

it removed all the trigger which got added for PromoteAction,So add the trigger which was not supposed to be removed,

 

MQL>mod policy "Formal Release" state Review add trigger Promote Check emxTriggerManager input TypeAllDRToolsV6RunCheckTrigger;

Confirm by print query,

2. Add the drV6tools Policy State Check Trigger Object ➣

•Add a suitably named 3dxtools Folder, or browse to an existing one.
•Select the Folder Content tab.
•Click on Actions and select Create DRLTools Type Check Trigger...

oThis will open the DRLTools Type Check Trigger create form.
•Set the following values:

{Type Name} should be set to the ENOVIA type of the object which the policy is assigned and the trigger is running on.

{Policy Name} should be set to the ENOVIA policy name.
