Copy Values Element MQL Selectable


Contents

The DRLTools Copy Values Element MQL Selectable allows you to define MQL select statement that can be executed against the current user, current object, parent, relationship, etc. The system uses the Source value to retrieve the required object and the runs the MQL select statement defined in Value against it.

 

To add a Copy Value Element Object Attribute object:

•Select the Copy Value Elements tab on the Copy Values Target Attribute.
•Click Actions.
•Select Create Copy Value Element MQL Selectable...

oThis will open the DRLTools Copy Value Element MQL Selectable create form.
•Set the following values:

Attribute Name

Value

Type

DRLTools Copy Value Element MQL Selectable

Name

{A suitable identifier}

Revision

{Any suitable value}

Source

This defines the source value. Depending on what is selected in this value drives the value that should be entered into the Value attribute.

Source Options

Description

object

Retrieve the MQL selectable from the current context object.

user 

Retrieve the MQL selectable from the current user object.

relationship 

Retrieve the  MQL selectable from the current context relationship.

parent

Retrieve the  MQL selectable from the current context parent object.

from 

Retrieve the  MQL selectable from the object on the from end of the current relationship

to

Retrieve the  MQL selectable from the object on the to end of the current relationship.

file

Retrieve the value  or MQL selectable from the current file object. NOTE: this value is only usable when 3dxtools is running the Copy Values Activity in the context of a file operation. i.e. Export Target Directory. The Value attribute must be set to either 'File Name', 'Format', 'File Extension', 'File Name Without Extension', or an MQL selectable on the file object.

Value

This value defines the MQL Select statement that should be run from the object defined in the Source value. It must be a valid MQL selectable, e.g. attribute[Part Weight].unitvalue[kg]
