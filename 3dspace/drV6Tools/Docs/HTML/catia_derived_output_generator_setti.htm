Derived Output Generator Settings


Contents

The Derived Output Generator Settings object define how the system should handle any files produced as a result of running the batch process defined in the CATIA Batch Processor Settings object. For instance if the batch process is setup to run the PLMBatchPrint process to produce a PDF file, then this object tells the system what to do with it, including the type, name and revision patterns to use, the policy and the make up of the intended output file name. It is also possible to perform more complex attribute mappings to the derived output object using the Copy Values system, this will allow you to map values from the source object to the created derived output object simply by associating Copy Values Target Attribute objects to the Target Attributes tab.

To add a Derived Output Generator Settings object:

•Select the Derived Output Settings tab on the CATIA Batch Processor object.
•Click Actions.
•Select Create DRLTools Processor Derived Output Generator Settings...

This will open the Processor Derived Output Settings create form.

•Set the following values:

Derived Output Object Uses Common Document Model

{TRUE or FALSE}

This defines if the derived output uses the common document model or not. If set to TRUE the system will check in the file and create all the required version objects. If set to FALSE the system will just check the file into the derived output business object.

{TRUE or FALSE}

If the Derived Output Object Uses Common Document Model  is set to TRUE then this value defines if the derived output should be versioned when a new one is checked in.

DO_NOT_REVISE_DERIVED_OUTPUT

{TRUE or FALSE}

This defines if the system should revise the Derived Output object from a previous revision and link the new revision to the new revision of the source object. By default this option is TRUE but if the Derived Output type is not revisable then you must set this option to FALSE.

EXPECT_MULTIPLE_SHEET_FILES_OUTPUT

{TRUE or FALSE}

This defines is the system should expect this batch operation to produce multiple files one for each sheet. If this is the case then the system will attempt to add the sheet name (which is added to the produced batch job output files) to the end of the file name.

CONNECT_DERIVED_OUPUT_TO_3DSHAPE

{TRUE or FALSE}

This defines if the system should, if the type is a VPMReference (i.e. Product), connect the created derived output to the child 3dshape object. By default this is set to TRUE as its a requirement of the OOTB schema.
