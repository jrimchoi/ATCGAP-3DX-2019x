Macro Values


Contents

Macro Values are a simple name/value string substitution mechanism within 3dxtools, and provide a similar functionality to the Macro values used in ENOVIA. Many of the 3dxtools Activities allow you to use Macro Values within their configuration to allow a greater flexibility. Using the Macro Values it is very easy to substitute configuration values with values from the current object, current user, ENOVIA RPE and many other options.

The Macro Values syntax is ${key} or ${key~modifier} where key is the macro key setting to substitute the text with, and modifier can optionally modify or format the value retrieved by key.  Examples of modifications are changing the text to upper case, only retrieving the left two characters, or formatting a date value to display in a specific date-time format.

Macro Value

Example

Notes

${Activity Specific Value Name}

${objectURL}

Each 3dxtools Activity can have its own activity specific settings depending on the context that the activity is running in. For instance the ${objectURL} is a specific value used in the Send Email Activity.

If the Activity Specific Values Names collection doesn't contain the value defined then the system tries to retrieve the value from the current context object (see below).

${Attribute Name}

${Title}

If the Activity Specific Value Names collection does not contain Attribute Name value then the system will try and get the value from the current object. In this case it can be any basic or attribute name from the current context object.

${object:Attribute Name}

${object:Title}

To force the system to use a value from the current context object use the object: switch value. You can then define any basic or attribute name from the current context object.

${lookup:Lookup Name}

${lookup:CALCULATE_RESEND_DATE}

Use a value from the 3dxtools lookup values list.

${mqlSelectable:MQL Select Statement}

${mqlSelectable:relationship[NCR 

Raised Against].to.name}

This enables you to define any valid MQL selectable run against the current context object.

${mqlCommand:MQL Command}

${mqlCommand:eval expression 

'count TRUE' on temp query bus 

Documents * *}

This replaces the macro results with the results from the MQL command.

${parent:Attribute Name}

${parent:Title}

Use a basic or attribute value from the current context parent object.

${relationship:Attribute Name}

${relationship:Quantity}

Use a basic or attribute value from the current context relationship object.

${to:Attribute Name}

${to:Title}

Use a basic or attribute value from the current context relationship to end object.

${from:Attribute Name}

${from:Title}

Use a basic or attribute value from the current context relationship from end object.

${source:Attribute Name}

${source:Name}

Use a basic or attribute value from the source object for the activity.

${requestParameter:Request Parameter Name}

${requestParameter:suiteKey}

Use a request parameter from the current URL.

NOTE: this value can ONLY be used if the 3dxtools activity is initiated using a Command Action Trigger

${rpe:RPE Variable Name}

${rpe:MATRIXHOME}

Use a value from the ENOVIA RPE. See the ENOVIA documentation on what values are available to use from the RPE.

${userBusinessObject:Attribute Name}

${userBusinessObject:First Name}

Use a basic or attribute value from the current users business object.

${userAdminObject:Property Name}

${userAdminObject:Site}

Use a property from the current users admin object.

${jpo:jpoProgramName.jpoMethodName}

${jpo:get3DXMappings.getJPOValue}

Modifier Option

Example

Notes

${key~toupper}

${name~toupper}

${object:Title~toupper}

 

Sets the retrieved value to upper case.

${key~tolower}

${name~tolower}

${object:Title~tolower}

 

Sets the retrieved value to lower case.

${key~left:number of characters}

${object:name~left:3}

Retrieves a set number of characters from the left hand side of the value.

${key~right:number of characters}

${object:name~right:3}

 

Retrieves a set number of characters from the right hand side of the value.

${key~mid:start:number of characters}

${object:name~mid:3:5}

Retrieves a set number of characters from the middle of the value.

${key~split:split character:index}

${object:name~split:-:0}

Splits the value into a list using a specific character and then retrieves one of the values from the resulting list.

split character defines the character to use to split the base value with.

index defines the index number for the position in the resulting list to retrieve the value from.

${key~trimleft:number of characters}

${object:name~trimleft:3}

Removes a set number of characters from the left hand side of the value.

${key~trimright:number of characters}

${object:name~trimright:3}

Removes a set number of characters from the right hand side of the value.

${key~removeDuplicates:split character:separator}

${mqlSelectable:to[Vaulted Objects]

.from.name~removeDuplicates:|:, }

Splits the value into a list using a specific character, removes any duplicate values and then joins the remainder back together using the separator between each value.

split character defines the character to use to split the base value with.

separator defines the string value to use to join the values that should be used to join them back together.
