DRLTOOLS Route Setting


Contents

 This object is used to add settings for route node task event. This will be linked to Route Template and Route Template Task as 3dxtools Setting.

For route template, it will be used to configure operations on the Start or End of a route.

For route template task, it will be used to configure the specific operations based on action taken by task assignee. Following are the basic operations configured using route setting to the task:

1.Promote/Demote the route content based on user action (Approve, Reject etc.)
2.Attribute mapping to the route content. For example, task title, user comment and assignee name can be mapped to content using dependant activities.
3.Ability to select assignees for the next node to expedite the approval process.

 

drlSettings – multiline string attribute

 

below shows an example of the XML setting that this attribute should contain:

 

 <settings>

         <chooseUsers>

         <chooseUsersForAllTasks>

         <showOnlyEditableTasks>

         <showTaskSequence>

         <chooseUsersForTasks>

         <task optional=””>

         <task canNotChooseSelf =””>

         </chooseUsersForTasks>

         <chooseUsers>

         <promoteAttachedObjects>

         <promote type=""  policy="" fromState=”” toState="">

         <promote type=""  policy="" fromState=”” toState="">

         </promoteAttachedObjects>

         <demoteAttachedObjects>s

         <demote type=""  policy="" fromState=”” toState="">

         <demote type=""  policy="" fromState=”” toState="">

         </demoteAttachedObjects>

 </settings>

 

Example:

 <settings>

 <chooseUsers>

 <chooseUsersForAllTasks>false</chooseUsersForAllTasks>

 <showOnlyEditableTasks>false</showOnlyEditableTasks>

 <showTaskSequence>true</showTaskSequence>

 <chooseUsersForTasks>

 <task canNotChooseSelf="true">CAD Checker</task>

 <task canNotChooseSelf="true">Engineering Checker</task>

 <task optional="true" canNotChooseSelf="true">Lead Engineer Approval</task>

 <task>Final Approval</task>

 </chooseUsersForTasks>

 </chooseUsers>

 </settings>

 

 

chooseUsers: This setting has below mentioned settings to configure task assign process.

 

chooseUsersForAllTasks: This setting can be true or false. If true then during creation of route, user must assign users for all tasks and it is mandatory to assign users to proceed further. If false, chooseUsersForTasks setting will drive the process.

 showOnlyEditableTasks: This setting is used to show or hide the non-editable (disabled) tasks on assign page. If true, only editable tasks are displayed in Assign page. If false, all the tasks are displayed in assign page and not-editable tasks assignee selection will be disabled.

 showTaskSequence: This setting is used to show or hide task sequence number in assign page. If it is set as true, sequence numbers are displayed, if false sequence numbers are hidden in assign page.

 chooseUsersForTasks: Under this setting, user can define tasks for which assignees to be selected. If tasks are not mentioned, then based on the roles, user must accept the task.

Under this setting, there are following sub settings.

 

 canNotChooseSelf: This setting is used to allow/restrict self-assignment of tasks during assignment process.  If it is true, the context user cannot assign the tasks to self. If false, the task can be assigned to self.

This is an optional setting and default value is false.

 

Optional: This setting is used to make assignment mandatory/non-mandatory during assign process. If true, assignee selection can be skipped to the task to move further.

This is an optional setting and default value is false.

 

 i.e.

 <chooseUsersForTasks>

 <task canNotChooseSelf="true">CAD Checker</task>

 <task canNotChooseSelf="true">Engineering Checker</task>

 <task optional="true" canNotChooseSelf="true">Lead Engineer Approval</task>

 <task>Final Approval</task>

 </chooseUsersForTasks>

 promoteAttachedObjects: This setting is used to defined the policy (Mandatory), type, from state and to state of contents of Routes. During the operation based on the settings defined, system will filter out the contents of route and perform promote actions.

 

If fromState is defined, system will check the current state of content. If matches, then object is promoted to toState if mentioned or else it will promote to next state.

If settings contains toState mentioned, contents will be promoted to toState.

If fromState is mentioned along with toState, the if fromState matched with contents current state, then only content will be promoted to toState.

 

i.e.

 <settings>

 <promoteAttachedObjects>

 <promote type="DRL DOCUMENTS" policy="Formal Release" fromState="Draft" toState="WIP"/>
 <promote policy="Default Document"/>
 <promote type="DRL DOCUMENTS" policy="Standard Release" toState="WIP"/>
 <promote type="Engineering Document" policy="Customer Release" fromState="Draft"/>

 </promoteAttachedObjects>

 </settings>

 

 demoteAttachedObjects: This setting is used to defined the policy (Mandatory), type, from state and to state of contents of Routes. During the operation based on the settings defined, system will filter out the contents of route and perform demote actions.

 

If fromState is defined, system will check the current state of content. If matches, then object is demoted to toState if mentioned or else it will promote to next state.

If settings contains toState mentioned, contents will be demoted to toState.

If fromState is mentioned along with toState, the if fromState matched with contents current state, then only content will be demoted to toState.

 

i.e.

 <settings>

 <demoteAttachedObjects>

 <demote type="DRL DOCUMENTS" policy="Formal Release" fromState="Review" toState="WIP"/>
 <demote type="DRL DOCUMENTS" policy="Customer Release" fromState="Review" toState="WIP"/>

 </demoteAttachedObjects>

 </settings>
