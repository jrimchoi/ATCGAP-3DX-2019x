Validate Route Content At Start


Contents

When creating approval route you can select multiple documents. There should be a validation to allow documents which satisfy certain conditions to the approval process. This should be done before route creation.

Following are the basic validations configured using action activity:

•Exclude documents with Default Document policy in DMS approval process
•Exclude documents which are already released or at obsolete state

 

1. Business Object and Execution  ➣

Name

Description

Route Setting

DRLTools Route Setting_UsersForAllTasks_1

Trigger Object

DRLTools Type Check Trigger_Route Setting Validation Placeholder_1. Details at Validation Objects

This is placeholder object which allow us to create actions to be executed to validate route content.

Action Activity

DRLTools Action Activity_ValidateRouteContentAtStart_1. Details at Validation Objects

Execution

For each selected document of DOCUMENTS type system will run check condition for state. If state is Released or Obsolete operation will be blocked and redirected to validation report page.

Failure Message

If route content does not satisfy the defined criteria then system will block the operation and redirect control to validation report page
