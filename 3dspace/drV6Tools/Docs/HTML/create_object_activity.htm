Create Object Activity


Contents

The 3dxtools Create Object Activity provides a simple mechanism to allow you to dynamically create new objects using details and values from a source object. The activity lets you define if it should be created as a brand new object or as a clone of the current object (if set to clone the activity will copy all the current values and files across to the new object). Once the new object has been created the activity can also run Copy Values and also copy defined relationships from the source object to the newly created one.

 

Configuration and Settings

To setup a 3dxtools Create Object Activity:

1. Add a drV6tools Create Object Activity object ➣

•Select the Dependant Activities tab on either a top level 3dxtools Trigger action object or another 3dxtools Activity object.
•Click Actions.
•Select Create DRLTools Create Object Activity...

oThis will open the DRLTools Create Object Activity create form.
•Set the following values:

Use Shadow Agent

TRUE

or

FALSE

TRUE (ticked) means the system will use the background system super user to create the object. This is typically needed as usually standard users do not have the required permissions to carry out the operation.

FALSE (unticked) means the system will not use the background system super user to create the object.

Use Shadow Agent User

{Any shadow agent user name}

or

${SiteUserAgent}

or

{Leave Blank}

In some situations you may need to use a different shadow agent to carry out the create object operation. This value allows you to enter the user name or the shadow agent to use. The user must exist, be granted full permissions and also be assigned the standard system shadow agent password.

If you need to use a site specific user agent (for instance to ensure the file operations only occur on the users site) you can use the macro value ${SiteUserAgent} this will search the system for a user with a symbolic name of person_{Site Name}UserAgent

If left blank the system will use the 'User Agent' user.
