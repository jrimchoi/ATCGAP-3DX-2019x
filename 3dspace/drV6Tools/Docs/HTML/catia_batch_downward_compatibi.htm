Example CATIA Batch Downward Compatibility


Contents

Below is an example XML generated for the CATIA batch downward compatibility processor to generate a CATIA V5 data from a CATIA 3DEXPERIENCE drawing, part or assembly. With the output being a commpressed zip file of the CATIA V5 data checked in to 3DEXPERIENCE as a derived output object linked to the CATIA 3DEXPERIENCE object processed.

The sections highlighted in blue are specific to the configured environment connection and user details.

The sections highlighted in light blue are where Macro Values are being used to populate the data based on the object being executed upon.

The following tags contained within the XML marked in orange below are variables which control the output format

Optional Setting Name

Value

Notes

CATDWC_targetedVersion

{CATIA V5 Version/Release}

This defines the output version of the CATIA V5 data created. e.g. CATIAV5-6R2019.

CATDWC_3DRep_AsSpec

{YES or NO}

This defines the Geometrical Conversion Option. YES = Convert Part as Specification, NO = Convert Part as Result.
