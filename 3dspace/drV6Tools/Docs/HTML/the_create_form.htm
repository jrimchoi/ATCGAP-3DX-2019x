The Create Form


Contents

  This page will start the approval route process based on the settings defined in DRLTools Initiate Route object. From the settings system willl search Route Templates to display in the dropdown and also load the header, label from the settings.

 

The form below will be displayed in the sidepanel:

 

 

The user is presented with a form with the following elements

 

Create Form Header Text: This header text will be loaded from the <createFormHeaderText> XML setting value.

 

Template Label: This label text will be loaded from the <createFormTemplateLabelText> XML setting value.

 

Template List: This is a list of all Route Templates available for the user to select from based on the settings <routeTemplateRelationship> and <routeTemplateName>.

 

If the setting <routeTemplateRelationship> is defined, then this is the name of the relationship to use to find any linked Route Template objects attached to the currently selected business objects. If no route is found, then the message that should be displayed to the user can be found in the settings value <noRouteFoundMessage>. If multiple objects are selected then only the Route Templates that are associated with all selected objects should be displayed. If no Route Templates matching all objects are found then the error message defined in <noMatchingRouteFoundMessage> should be displayed.

 

If the setting <routeTemplateName> is defined then this is the name value to use to query the database for Route Templates with a matching name. If no route is found, then the message that should be displayed to the user can be found in the settings value <noRouteFoundMessage>.

 

Route Description: This is a simple multiline field that allows the user to enter text that will be mapped into the new route’s description value.

 

The user is also presented with two buttons:

 

Create: Calls the create form and at this point creates the new route and assigns the tasks to the roles

 

Close: Closes the form without making any changes
