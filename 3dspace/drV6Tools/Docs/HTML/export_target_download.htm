Export Target Download


Contents

The 3dxtools Export Target Download object provides the files and metadata selected as part of the parent Export Activity as an HTTP file download to the current users ENOVIA web session. This Export Target can only be used when the 3dxtools is run from a Command Action Trigger.  

The system will by default use the 3dxtools XML export format to output the metadata of the selected exported objects. The system provides the ability to parse the XML file through an XML translation process by enabling the administrators to define a standard XML EXtensible Stylesheet Language Transformation (XSLT) setting. Details on XSLT can be found here: XSLT Tutorial.

If the Export Activity results in multiple files then they will be added into a single ZIP file for download.

 

To create an Export Target Download object:

•Select the Export Targets  tab on the Export Activity object.
•Click Actions.
•Select Create Export Target Download...
oThis will open the DRLTools Export Target Download create form.
•Set the following values:

File Name

{File Name}

This defines the file name to use for the outputted XML metadata file containing all the metadata of the selected exported objects. If left blank the system will use the source object name.

Import XSL Source

{A Valid XML EXtensible Stylesheet Language 

Transformation (XSLT) setting}

This defines the XSL Transformation that should be applied to the XML metadata before it is saved to file. Using this value it is possible to configure the system to translate the XML into whatever format is required. Details on XSLT can be found here: XSLT Tutorial.

If the value is left blank then no translation will be applied and the default 3dxtools export XML format will be used.

Settings

{Additional Settings Values}

This setting defines the additional settings values. These are all optional and can be used on a as required basis.

ONE_OBJECT_PER_FILE={TRUE or FALSE} If set to TRUE the system will create a separate meta-data XML file for each object being exported.

SORT_ATTRIBUTES={TRUE or FALSE} If set to TRUE the system will sort the exported attributes by

putting the attribute names in alphabetical order.

 

SORT_RELATIONSHIPS={TRUE or FALSE} If set to TRUE the system will sort the child relationship objects

using the relationship name and the type, name and revision of the target object as the key to sort.

 

METADATA_FILE_NAME={File Name} If set this will be used for the file name for the meta-data xml file. 

This value will accept Macro Values.
