Send Email Activity


Contents

The 3dxtools Send Email Activity allows you to notify users of events triggered in ENOVIA by driving the inbuilt ENOVIA email/icon mail functionality. The activity can be attached to any operation trigger event or standard web UI command. The message subject and body can use Macro Values to add attributes from the current object, user, time stamp or URL request parameters.

 

Configuration and Settings

To setup a 3dxtools Send Email Activity follow the steps defined below.

1. Add a drV6tools Send Email Activity object ➣

•Select the Dependant Activities tab on either a top level 3dxtools Triggers action object or another 3dxtools Activities object.
•Click Actions.
•Select Create DRLTools Send Email Activity...

oThis will open the DRLTools Send Email Activity create form.
•Set the following values:

This defines the ENOVIA user name of the person(s) you wish to send the email message to. It can be fixed values or Macro Values.

Multiline Arg1

{Email message text to include for single object}

If the settings xml value <sendMultipleObjectsInOneEmail>  is set to true then this setting will be used. It defines the text that should be used for each object that is selected to be sent in a single email. It can use Macro Values. The values are concaterated into a single value which can then be used in the "Message" setting above to place the value in the overall email message text. This value is ${perObjectMessageText}

 

 

XML Send Email Activity Settings

The email settings define the extended settings for the Send Email Activity

<EmailSettings>
    <format>html</format>
    <to>
      <recipient>Customer Address</recipient>
      <recipient>${object:Customer}</recipient>
      <recipient source="role">Technical Writer</recipient>
    </to>
    <cc>
        <recipient source="group">Shadow Agent</recipient>
        <recipient source="person">admin_platform</recipient>
    </cc>
    <bcc>
        <recipient source="person">Test Everything</recipient>
    </bcc>
    <smtpServer>localhost</smtpServer>
    <defaultSender>test@test.com</defaultSender>
    <userAsSender>true</userAsSender>

    <sendMultipleObjectsInOneEmail>true</sendMultipleObjectsInOneEmail>
    <attachCurrentObjectFiles>true</attachCurrentObjectFiles>
    <attachCurrentObjectFilesWithExtensions>doc,docm,docx</attachCurrentObjectFilesWithExtensions>
    <attachDerivedOutputFiles>true</attachDerivedOutputFiles>
    <attachDerivedOutputFilesWithExtensions>pdf</attachDerivedOutputFilesWithExtensions>
    <derivedOutputType>Derived Output</derivedOutputType>
    <derivedOutputRelationship>Derived Output</derivedOutputRelationship>
    <embedImages>true</embedImages>
    <embedImagesWithFileExtensions>jpg,png,gif</embedImagesWithFileExtensions>
</EmailSettings>

 

•The table below describes each of the XML Elements

XML Element

Notes

<format>

This defines the format for the email message. If set to text then the system will interpret the value in the Message attribute as plain text.

<format>text</format>

If set to html then the system will interpret the value in the Message attribute as HTML.

<format>html</format>

The default is plain text.

<to>

The list of <recipient> objects to send the email To.

<to>
  <recipient>Customer Address</recipient>
  <recipient>${object:Customer}</recipient>
  <recipient source="role">Technical Writer</recipient>
</to>

 

Each recipient defines either a single or list of users or email addresses. These can be pulled from either a specific Person, people assigned to Groups or Roles or people or email addresses defined on business object attribute values. To control this the source XML attribute is used.

To set to a specific Enovia user:

<recipient source="person">User Name</recipient>

To set to the users assigned to an Enovia role:

<recipient source="role">Role Name</recipient>

To set to the users assigned to an Enovia group:

<recipient source="group">Group Name</recipient>

To set to the users or email addresses defined inside the current objects business object attribute value use source="value". When setting to this method you can either define an attribute name or a Macro Values.

<recipient source="value">Attribute Name</recipient>

<recipient source="value">${object:macro}</recipient>

source="value" also forms the default for the recipient object.

<recipient>Attribute Name or Macro Value</recipient>

    below shows an example using an attribute or macro value 

 

    <to>
      <recipient>Customer Address</recipient>
      <recipient>${object:Customer}</recipient>
    </to>

 

    

 

 

<cc>

The list of <recipient> objects  to CC the email to.

<bcc>

The list of <recipient> objects to BCC the email to.

<smtpServer>

The address of the SMTP server to use to send the email. If not defined the the system will try to read the value used for Enovia defined in the enovia.ini file.

<smtpServer>localhost</smtpServer>

<defaultSender>

The default email address to send the email from.

<defaultSender>test@test.com</defaultSender>

<userAsSender>

This defines if the email should be sent as if from the current user. If set to true then the users fullname and email address will be used (if the user has no email address defined then the default sender value will be used instead. The default for this value is false.

<userAsSender>true</userAsSender>

or

<userAsSender>false</userAsSender>

<sendMultipleObjectsInOneEmail>

This defines when command is run from a workspace folder and the user selects multiple objects if the system should combine all the objects into one email or send one email per object. The default is false.

<sendMultipleObjectsInOneEmail>true</sendMultipleObjectsInOneEmail>

or

<sendMultipleObjectsInOneEmail>false</sendMultipleObjectsInOneEmail>

<attachCurrentObjectFiles>

This defines if the files on the current business object should be attached to the email

<attachCurrentObjectFiles>true</attachCurrentObjectFiles>

or

<attachCurrentObjectFiles>false</attachCurrentObjectFiles>

<attachCurrentObjectFilesWithExtensions>

If attachCurrentObjectFiles is set to true then this value will let you restrict the files to be attached to only ones with specific file extensions.

<attachCurrentObjectFilesWithExtensions>doc,docm,docx</attachCurrentObjectFilesWithExtensions>

<attachDerivedOutputFiles>

This defines if the files on any derived output objects linked to the current business object should be attached to the email.  The default for this value is false.

<attachDerivedOutputFiles>true</attachDerivedOutputFiles>

or

<attachDerivedOutputFiles>false</attachDerivedOutputFiles>

If set to true then you must define derivedOutputType and derivedOutputRelationship.

<attachDerivedOutputFilesWithExtensions>

If attachDerivedOutputFiles is set to true then this value will let you restrict the files to be attached to only ones with specific file extensions.

<attachDerivedOutputFilesWithExtensions>pdf,jpg</attachDerivedOutputFilesWithExtensions>

<derivedOutputType>

This defines the derived outputs business type.

<derivedOutputType>Derived Output</derivedOutputType>

<derivedOutputRelationship>

This defines the relationship type that is used to connect the current object to the derived output.

<derivedOutputRelationship>Derived Output</derivedOutputRelationship>

<embedImages>

This defines if the system should embed images inside the email. It should only be used if the format is set to html. The default for this value is false.

<embedImages>true</embedImages>

or

<embedImages>false</embedImages>

If set to true then you must check the image files into template business objects within ENOVIA.

These business objects should then be connected to the Send Email Activity under the New From Template tab.
