Type Check Trigger


Contents

The 3dxtools Type Check Trigger is used to connect an check event on a Type in ENOVIA with a series of 3dxtools Activities to execute when the event fires. Some examples of the actions you can perform using these triggers are:

•Use the Change Document Name action when changing document name.
•Use the Revise Document actions when revising the document.

 

Configuration and Settings

To setup a 3dxtools Type Check Trigger:

1. Add the Event Trigger to the Type in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Type you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the type you wish to use by querying like  below,

  MQL>list type ${TypeName};

 

MQL will list the type name.

•On the Triggers for that type find the ENOVIA trigger you wish to run the 3dxtools Activities on.
•To find the triggers which are already existing on type write the below query,

 MQL>print type ${TypeName} select trigger;

  This will print already existing triggers for the mentioned type including both inherited triggers(inherited from parent type) and immediate triggers(added on this type)

  Example: Print type "DRL Documents" select trigger;

 

The MQL result will list the triggers which are already added

    trigger = ChangePolicyCheck:emxTriggerManager(TypeAllDRToolsV6RunCheckTrigger)

    here,the trigger event is ChangePolicy and trigger type is Check (for more details on events and trigger type refer below table).

    for the ChangePolicy already a trigger named TypeAllDRToolsV6RunCheckTrigger has added.

   MQL>print type ${TypeName} select immediatetrigger;

  This will print already existing triggers for the mentioned type which are directly added on the type.

•To add new trigger to the type
•Edit the trigger.
•Set the following values:

Name

Value

Notes

Action

emxTriggerManager

For details on emxTriggerManager please see the standard ENOVIA documentation.

Input

Append to any existing value

TypeAllDRToolsV6RunCheckTrigger

TypeAllDRToolsV6RunCheckTrigger runs the dependant 3dxtools Activities immediately.

Trigger_Type

Name of the Trigger Type :Check

List of Events: Check,Action,Override

Event_Name

Name of the Event:Create

Modify,Create,Revise,etc..please see the standard ENOVIA documentation for more events

    to add new input to the event(ChangePolicy) which already exits for the trigger type(Check),

Syntax:

MQL>mod type ${Type_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Note:If immediate trigger already exists then append the new trigger and existing trigger with comma separator,

Example:

MQL>mod type "DRL Documents" add trigger ChangePolicy Check emxTriggerManager input TypeAllDRToolsV6RunCheckTrigger,TypeAllDRToolsV6RunCheckTrigger_Deffered;

Confirm the trigger added  from the below query:

      MQL>print type "DRL Documents" select triggers;

     

•Remove trigger:

 To remove trigger from the type mention the event and trigger type, it removes all the trigger which got added

     Before removing the trigger ,make a note of trigger which should not be removed  for the event,so that it can be added later

     MQL>mod type "DRL Documents" remove trigger ChangePolicy Check;

   

    The specified trigger type is removed from the listed event except the trigger which got inherited from parent.

•Confirm the trigger added  from the below query:

   MQL>print type ${TypeName} select triggers;

   

    it removed all the trigger which got added for ChangePolicy,So add the trigger which was not supposed to be removed,

    MQL>mod type "DRL Documents" add trigger ChangePolicy Action emxTriggerManager input TypeAllDRToolsV6RunCheckTrigger;

   

2. Add the drV6tools Type Check Trigger Object ➣

•Add a suitably named 3dxtools Folder, or browse to an existing one.
•Select the Folder Content tab.
•Click on Actions and select Create DRLTools Type Check Trigger...

oThis opens the DRLTools Type Check Trigger create form.
•Set the following values:

Attribute Name

Value

Notes

Type

DRLTools Type Check Trigger

This should already be set for you

Name

{Operation Name}_{Type Name}

{Operation Name} should be set to the trigger in ENOVIA you wish to run the activities from (for instance ChangeName, ChangePolicy, Checkin, etc)

{Type Name} should be set to the ENOVIA object type the trigger is being run on.

{any suitable value}

{any suitable value}

4. Add dependant drV6tools Activity objects ➣

•From the Dependant Activities tab of the Type Check Trigger, add any 3dxtools Activities you wish to run from this event.
