Route Setting Actions


Contents

A series of actions can be configured with route setting objects. These actions will be used to validate route content or task details during or before starting any approval process.

To configure a route setting actions you need to define action activity name criteria in respective route setting object.

The validation system will search for the action and execute the checks. If there is any failure then the operation will be blocked. An easily readable failure message will be displayed to the user.

The message can be either page redirect to validation report in interactive UI or notice in trigger events.

Name

Description

Validate Route Content At Start

When creating approval route you can select multiple documents. There should be a validation to allow documents which satisfy certain conditions to the approval process. This should be done before route creation

There should be a validation to validate documents which satisfy certain conditions before task approve or reject
