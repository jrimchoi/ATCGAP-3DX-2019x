Policy State Action Trigger


Contents

The 3dxtools Policy State Action Trigger is used to connect an action event on a Policy State in ENOVIA with a series of 3dxtools Activities to execute when the event fires. Some examples of the operations you can perform using these triggers are:

•Use the Export Activity when an object is released to export the metadata and files to an external system.
•Use the Send Email Activity to notify users when an object is released.
•Use the Create Queue Object Activity to queue up the object to have the final released derived outputs (e.g. a PDF) generated.

 

Configuration and Settings

To setup a 3dxtools Policy State Action Trigger follow the steps defined below.

1. Add the Event Trigger to the Policy State in ENOVIA ➣

•In ENOVIA MQL find the ENOVIA Policy you wish to use.
•Go to Windows Start Menu ->mql ->click on "Start Mql.exe".
•set the context by writing below query,

  MQL>set context user creator;

•find the policy you wish to use by querying like  below,

  MQL>list policy ${Policy_Name};

 

    MQL will list the policy name.

•On the Policy find the state you wish to run the trigger from.

 MQL>print policy ${Policy_Name} select state ${State_Name.trigger};

 Example: MQL>print policy "Formal Release" select state[Review].trigger;

 

The MQL result will list the triggers which are already added

    trigger = PromoteAction:emxTriggerManager(PolicyAllStateAllDRToolsRunActionTrigger)

    here,the trigger event is Promote and trigger type is Action (for more details on events and trigger type refer below table).

    for the PromoteAction already a trigger named PolicyAllStateAllDRToolsRunActionTrigger has added.

•To add new trigger to the Review state,
•Edit the trigger.
•Set the following values:

Name

Value

Notes

Action

emxTriggerManager

For details on emxTriggerManager please see the standard ENOVIA documentation

Input

Append to any existing value:

PolicyAllStateAllDRToolsV6RunActionTrigger

or

PolicyAllStateAllDRToolsV6RunActionTrigger_Deferred

PolicyAllStateAlDRToolsV6RunActionTrigger runs the dependant 3dxtools Activities immediately.

PolicyAllStateAlDRToolsV6RunActionTrigger_Deferred waits until the current transaction scope has completed before running the dependant 3dxtools Activities.

Trigger_Type

Name of the Trigger Type :Check

List of Events: Check,Action,Override

Check: It performs an action before an event occurs.

Action: It performs an action by replacing an event.

Override:It performs an action after an event has occurred.

Event_Name

Name of the Event:Promote

Modify,Create,Revise,checkin.etc..please see the standard ENOVIA documentation for more events
•to add new input to the event(Promote) which already exits for the trigger type(Action),

Syntax:

MQL>mod policy ${Policy_Name} state ${State_Name} add trigger ${Event_Name} ${Trigger_Type} emxTriggerManager input ${Input_Value};

Note:If  trigger already exists for ModifyAction then append the new trigger and existing trigger with comma separator,

     Example:mod policy "Formal Release" state Review add trigger Promote Action emxTriggerManager input PolicyAllStateAllDRToolsV6RunCheckTrigger,PolicyAllStateAllDRToolsV6RunActionTrigger_Deferred;

   

• Confirm the trigger added  from the below query:    

    MQL>print policy "Formal Release" select state[Review].trigger;

   

•Remove trigger:

  To remove trigger from the attribute mention the event and trigger type, it removes all the trigger which got added

  Before removing the trigger ,make a note of trigger which should not be removed for the event,so that it can be added later

 MQL>mod policy "Formal Release" state Review remove trigger Promote Action;

 

 MQL>print policy "Formal Release" select state[Review].trigger;

 

it removed all the trigger which got added for PromoteAction,So add the trigger which was not supposed to be removed,

 MQL>mod policy "Formal Release" state Review add trigger Promote Action emxTriggerManager input PolicyAllStateAllDRToolsV6RunCheckTrigger;

 

2. Add the drV6tools Policy State Action Trigger Object ➣

•Add a suitably named 3dxtools Folder, or browse to an existing one.
•Select the Folder Content tab.
•Click on Actions and select Create DRLTools Type Actions Trigger...

oThis will open the DRLTools Type Action Trigger create form.
•Set the following values:

Attribute Name

Value

Notes

Type

DRLTools Type Action Trigger

This should already be set for you

Name

{Operation Name}_{Type Name}_{Policy Name}_{Policy State Name}_{To State Name}

or

{Operation Name}_{Type Name}_{Policy Name}_{Policy State Name}

{Operation Name} should be set to the trigger in ENOVIA you wish to run the activities from (for instance Create, Modify, Checkin, etc)

{Type Name} should be set to the ENOVIA type of the object which the policy is assigned and the trigger is running on.

{Policy Name} should be set to the ENOVIA policy name.

{Policy State Name} should be set to the ENOVIA policy state name the trigger is assigned to.

{To State Name} this optional value is the state name to object is being promoted to. This is useful if the state has multiple paths coming out of it.

Revision

{any suitable value}

Description

{any suitable value}

Success Message

Not applicable

Failure Message

Not applicable

•From the Dependant Activities tab of the Type Action Trigger, add any 3dxtools Activities you wish to run from this event.
